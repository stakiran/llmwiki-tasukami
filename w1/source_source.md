# 「タスクソース」

chap3 の最終章。タスクを生み出すソース（源泉）の概念とその運用を扱う。

## 要約

- **[タスクソース](task_source.md)** = タスクを生み出す何らかのもの（特に言語化されたもの）
- 4 種類: 目標事項 / 維持事項 / 連想事項 / 発想事項
- 連想事項の運用には 3 つの方向性: **連想リスト**（トリガーリスト）/ [発散と収束](divergent_convergent.md) / 俯瞰のつくりこみ
- 仕事には **ソースアプローチ** と **プランアプローチ** の 2 通り
- ベストは「ソース → プラン」の切り替え。切り替えタイミングが [SP 臨界点](sp_threshold.md)

## 章で導入される主な概念
- [タスクソース](task_source.md) — タスクの源泉とその 4 種類
- [発散と収束](divergent_convergent.md) — 動的な連想事項
- [SP 臨界点](sp_threshold.md) — ソースアプローチ → プランアプローチの切り替えタイミング

## 引用しておきたい箇所
- 「**連想に頼るのが苦手な人の大半はノイズに潰されているからではないか**」
- 「**発散と収束は自分ひとりで集中して行える環境を確保できるかどうか**」
- 「複数人で行いがちですが、実は典型的な悪手」（発散と収束について）
- 「**最初はソースアプローチを行ない、どこかでプランアプローチに切り替える**」
