# ネットワーク型ノートツール

ノートとノートをリンクで繋ぐことで、ノート全体をネットワーク状にできるノートツール。近年台頭してきたパラダイム。

例: [Scrapbox](https://scrapbox.io/) / [Obsidian](https://obsidian.md/) / Roam Research / Logseq

## 仕組み

- ページ A で `[[B]]` や `[ ]` と書くと、A → B のリンクが生成される
- ページ B 側にメタ情報として「A から接続されている」が表示される（**バックリンク**）
- リンクをクリックして即座にジャンプできる

ウェブページがリンクで辿れるのと同じことを、個人の手元のノートで実現する。リンクも自分で紡いでいく。

## トピック指向

ネットワーク型ノートで使われる発想。

> 1-話題 1-ノート

タスクやプロジェクトのような分類しやすい単位ではなく、**話題（トピック）** ごとにノートを切る。

「炊き込みご飯ってめっちゃ美味いんだな」のような自由なノートを作れる。話題の単位は大きくても小さくても OK。

| パラダイム | 単位 | ノート数の目安 |
| --- | --- | --- |
| ダイアリーベース | 日 | 数百 |
| プロジェクトベース | プロジェクト | 数十 |
| **トピック指向**（ネットワークベース） | 話題 | 数千〜数万 |

数千〜数万のノートを破綻させずに扱えるのはデジタル + ネットワーク構造ならでは。

## 第二の脳

ネットワーク状のノートは **第二の脳** になりえる。

- 人間の脳、交通網、インターネット、人間関係 — ネットワークは馴染み深い構造
- 個人ノートに取り入れることで圧倒的な書きやすさと辿りやすさを実現
- リンクを辿って探索したり、リンクしている / されている周辺の情報を表示できる

> ただのダイアリーやプロジェクトでは到達できない高み

[LTM](ltm.md) はこの構造を前提とする。

## オンライン版とオフライン版

| | 例 | 特徴 |
| --- | --- | --- |
| オンライン | **Scrapbox** | 出来が良い。著者おすすめ。ただし通信環境・許可面の制約 |
| オフライン | **Obsidian** / Logseq | クラウド同期も別途付与可能。仕事で使いやすい |

著者の推奨: オンラインなら Scrapbox、オフラインなら[プレーンテキストエディタ](plain_text.md) で工夫。

## 無限キャンバスとの比較

**無限キャンバス**（[Miro](https://miro.com/) など）も大量の文脈を管理できるパラダイム。

| | ネットワーク型ノート | 無限キャンバス |
| --- | --- | --- |
| 重視するもの | 言語（テキスト） | 視覚・空間 |
| 構造 | リンクのネットワーク | 二次元配置 |
| 強み | 大量テキストの管理 | 多様コンテンツの空間配置 |
| タスク管理との相性 | ⭕ | 🔺 |

> 大量の文脈を管理できるパラダイムは、筆者の知る限りこの二つだけ

[LTM](ltm.md) のような言語ベースのタスク管理ではネットワーク型ノートに分がある。タスク管理は言語を扱うものであり、自分で言語化したものを扱う性質が言語重視のノートツールと噛み合う。

> 結局言語で文脈を残す（しかない）

[Ink & Switch の Embark](https://www.inkandswitch.com/embark/) でも、無限キャンバスを使っていても結局メモを書き、それを見て行動すると考察されている。

## ノートツールのデメリット: ビューが弱い
ネットワーク型ノートの欠点は **ビュー（見せ方）が弱い** こと。とくに [LTM](ltm.md) では、書いたタスクをどう俯瞰するかが主要トピックになる。

> ビューが弱いからといってノートを使うのをやめないこと

LTM をするならノートから離れない。ビューは別途補う（自作スクリプト、生成 AI などの活用）。

## アウトライナーとの違い
**アウトライナー**（WorkFlowy / Dynalist）はすべての情報を箇条書きで表現する別パラダイム。階層的な並び替えや折りたたみが柔軟だが、ネットワーク（横の繋がり）をつくる能力は弱い。LTM 用途では不向き。
