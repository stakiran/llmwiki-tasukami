# プロセスの管理

手順を管理すること。[5 つの管理対象](management_targets.md) の 1 つ。

タスク管理上は「手順」「手続き」として現れる。タスクの出来を確保し安定させるため、みんなこの通りに作業してね、とする。**再現性** とも言う。プロセスと手順は同義。

## プロセス活動と非プロセス活動

> プロセスは典型的なシャドータスクになりがち

タスク T の見積時間が 3 時間で、うちプロセスに従う部分が 1.5 時間なら、T の「プロセスに従う以外」は 1.5 時間しか残らない。

- **プロセス活動** — プロセスに従う部分
- **非プロセス活動** — それ以外

両者を捕捉できないと見積もりが甘くなり、どちらかが [シャドータスク](shadow_task.md) になる。

例: T が「アイデアを形にするのに 3 時間 + 採用までのプロセス 1.5 時間」なら 4.5 時間と見積もるべき。3 時間にしちゃうと非プロセスの 3 → 1.5 時間圧縮 = アイデアの質が落ちる、または時間外で捻出（シャドー化）。

逆もある。プロセスに慣れてくると麻痺して、自分達のプロセスにかかる時間を忘れがち。

管理過多もある:
- やり方なんて自由でいいのに無理やりプロセスを適用
- プロセス自体は必要だが細かすぎて時間がかかる
- 承認ステップでボトルネック発生

## 整理

| 区分 | 対処 |
| --- | --- |
| 必要なプロセス活動 | ちゃんとタスク化する（または含む形で見積もる） |
| 過剰なプロセス活動 | なくす。問題が出たら追加 |
| 非プロセス活動 | 通常のタスク |

> 要らないものはとりあえずやめて、それで様子を見てください

私たちはビジネス文脈でプロセスを定めたがるが、定めなくても行えるタスクは案外多い。「このプロセスは本当に必要なのか?」と自問する。

## プロセス原理主義

邪悪な管理者はプロセスでも顔を出す。「そのとおりに則っているか見ればいい」というかんたんなお仕事で済むのでしがみつきやすい。**プロセス原理主義**: プロセスは絶対的に正しいのだから当然、という信念。

法律と違ってプロセスは管理者がかんたんに追加でき、暴走化しやすい。善意に基づくこともある（「地獄への道は善意で舗装されている」）。

対処は進捗の管理者より困難:
- 管理者から離れる
- 表向きは従うふりをする
- 上位や結託で追放する
- 最初から管理の少ない仕組み（組織）をつくる

## まとめ
- プロセスの管理はシャドー化しやすい
- 必要・過剰・非プロセスの 3 区分。過剰をいかに削るかが重要
- 邪悪な管理者は法律的信念付きで現れ、対処は困難
