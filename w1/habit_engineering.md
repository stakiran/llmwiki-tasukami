# ハビットエンジニアリング (Habit Engineering)

良い [習慣](habit.md) を新しく増やしたり、悪い習慣を減らしたり、微妙な習慣の対処を決めたりすることの総称。著者の造語。

中核に **GSAAD ループ** がある。

## GSAAD ループ

習慣をいきなり採用したり取り除いたりはできない。日々少しずつ行動して、結果をチェックして微修正する地道な営みで勝ち取る。

| 段階 | 略 | 意味 |
| --- | --- | --- |
| **G**et | ターゲット | 標的となる習慣（行動）を定める |
| **S**et | 仕掛ける | ターゲットに干渉する仕掛けを施す |
| **A**ct | 行動する | 仕掛けをトリガーに行動する |
| **AD**just (Adjust) | 微調整 | 結果を踏まえて微調整 |

Adjust の行き先が Get / Set / Act の複数あるのが特徴。

### 1. Get — ターゲットを定める
扱いたい習慣を **「具体的な行動」のレベル** で定める。痩せたい・賢くなりたいといった願望ではなく、行動にフォーカス。

味気がなくやる気が湧かなくても OK。それでも「習慣にしたい」と思える行動でなければならない。なければハビットエンジニアリングは続かないので諦める（願望をブレイクダウンして行動に落とすところから）。

### 2. Set — 仕掛けを施す
意志の力だけでは毎回行動できない。**特定の条件で自動的に行動するような仕掛け** をつくる:

- **[動線](dousen.md) / [リマインド](remind.md)** — 前章で扱った仕掛け全般
- **ハビットスタッキング** — 新しい行動を「既存の習慣」にくっつける。既存習慣は欠かさず行うので、ついでに新しい行動もやりやすくなる。例: 歯磨きのそばに鼻毛カッターを置く
- **IF-THEN プランニング** — 「もし XXX が起きたら YYY をする」のルール化。[モットー](motto.md) と等価で向き不向きが分かれる
- **タスクリストやインボックスへの追記** — 普段使っているところに追記

本質は 2 つだけ:

1. 忘れないように気付かせる
2. 気付いた後に行動するよう **強度を高める**

強度の最たる例が [ヒューマンリマインドや雰囲気リマインド](remind.md) の「外部から働きかけてもらう」。逆にアスリートタイプは TODO リストに一行で行動できる。

> 自分が実際に行動に起こせる程度の強度を設定すること

### 3. Act — 仕掛けをトリガーに行動する
行動の中身、つまり **行動パラメーター（Action Parameter）** が重要。例（散歩）: 何分歩く、服装、持ち物、場所、雨の有無。

> 仕掛けた行動が習慣として定着するかどうかはパラメーターが適切かどうかで決まる

いきなり 3 時間散歩しようとしても続かない。まず 10 分だけ、雨の日はやらない、ショッピングモールに行く時にくっつける（雨パラメーターをなくす）など。

#### 着手ベース

> 重要なのは行動の質ではなく行動の回数

「100 点よりも 60 点を取れ」と同じ。**着手さえできたら進捗があった** とみなす考え方（**着手ベース**）。作業興奮もあるので、最初の腰を上げるところが最重要。

ただし軽くしすぎても効かない。「これから散歩します、と音読すれば OK」レベルだと「で?」となって先が続かない。「とりあえず玄関で靴を履いてみる」くらいなら続きやすい。

### 4. Adjust — 微調整する

| 結果 | 疑う対象 |
| --- | --- |
| **行動できた** | 行動パラメーターを調整してより快適に（Act） |
| **行動できなかった** | 仕掛け（Set）か行動パラメーター（Act）を疑う |
| **行動しなかった** | ターゲット（Get）か行動パラメーター（Act）を疑う |

**できなかった** = 何らかの事情（忘れ、疲労）でできなかった。
**しなかった** = できないわけじゃないが腰が上がらなかった。主因は (a) パラメーターがきつすぎる、(b) ターゲット自体が不適切（モチベ不足）。

> 解は「モチベが湧くまで諦めなさい」あるいは「別の、モチベが湧く行動をターゲットにしなさい」

## 定着とハビチュエーション

**定着（ハビチュエーション, Habituation）** = 良い習慣を一つ獲得 / 悪い習慣を一つ断ち切ること。

目安: 一ヶ月 / 二ヶ月 / 一年。一日も欠かさず管理に頼ることなくこなせる状態。体感で「そろそろ定着してきた」とわかる（GSAAD の仕掛けを無視し始める / 「もう仕組みに頼らなくてもできる」と感じる）。

## 撤収

定着確認後は GSAAD の仕組みや微調整を一切なくす（**撤収**）。

定着した習慣も稀に崩れることがあるが、一時的なら気にしなくていい。何回か続くなら点検する。

> 生活の過ごし方が慌ただしく変化する人は定着しづらい

主因は生活の変化（自分のキャパシティに入らなくなる）。**生活をできるだけ固定する** のが習慣化の近道（生活リズム）。

## 悪い習慣を減らす

> 「しない」を行動に落とすのは難しい

悪い習慣を減らすアプローチは 3 つ:

### 1. 妨害する
悪い習慣（の行動）を妨害する別の行動や仕組みを差し込む。例（寝る前飲酒）: 冷蔵庫内のお酒を寝る前に空にする、風味を損なうためにガムを噛む、歯磨きを寝る直前に変える。

直接直前に差し込めない場合は「妨害を行う仕組みを設置するための行動」を仕込む。妨害行動は習慣化する必要なし、ルーチンタスクや日課で OK。

### 2. 根本を排除する（[盤外戦](bangaisen.md)）
妨害だけでは追いつかない場合、悪い習慣のもとに直接干渉。例: 病気・依存なら通院、ストレス源の特定と排除、人間関係を断つ、寝不足の原因（スマホ）を変える。

直接働きかけよりも **距離を置く** 方が摩擦が少ない。スルーを試して、ダメなら腹をくくって直接、の二段階。

### 3. 撹乱（かくらん）する
**自身の生活環境を劇的に変える**。引っ越し、転職など。

> 人間が変わる方法は 3 つしかない: 時間配分、住む場所、付き合う人

いずれも環境を変えるレベルのインパクト。

## 微妙な習慣を倒す

> 良い習慣なのか悪い習慣なのか、どちらかであると決めること

例: 筋トレは筋力向上の面で良いが、免疫低下や体温変化で体調を崩しやすいので悪い。決めない限り認知資源を食い続ける。決めて生活してみて、影響を見て再判定すれば良い。

> 微妙な習慣をさっさと扱うフットワークは、ハビットエンジニアリングの成否を分ける鍵

微妙な習慣は **「悪い習慣候補」** とも呼べる。

## 捨て身 — 自律できない人向け

GSAAD は自力で仕込んで自力で行動する前提だが、特性上それが苦手な人もいる。

解はあまりない:

- Set の項で挙げた **環境や人など外部の力に頼った仕掛け**
- 撹乱（環境ごと変える）

> 職人の弟子入り、スポーツのクラブ活動、寮生活、合宿などでは生活を厳しくコントロールされ年単位で続くので、嫌でも習慣が身につく

これらは **荒療治**。本来、習慣はそれくらいの荒療治が必要。地道なハビットエンジニアリングができないなら、荒療治しかない = **捨て身**。
