# 「習慣」

[オルタスク](altask.md) のひとつ「習慣・日課」を本格定義する章。chap4 の 2 つ目。

## 要約

- 「継続的にこなす営み」を **日課 / ルーチンタスク / 習慣** の 3 つに分ける
    - 日課 = デイリーミッション、いつ達成してもいい。[Tracker](tracker.md) で管理
    - [ルーチンタスク](routine_task.md) = 頻度を持ったタスク。ツール上の概念。[Robot](robot.md) で管理
    - **[習慣](habit.md)** = 定期的に例外なく行う、定着した事柄
- 習慣には 3 種類: **良い習慣 / 悪い習慣 / 微妙な習慣**
- 習慣の意義は **行動の最大化**
- 習慣を扱う方法論が **[ハビットエンジニアリング](habit_engineering.md)** で、中核に **GSAAD ループ** がある
- 自律できない人向けに **捨て身**（環境ごと変える）

## 章で導入される主な概念
- [習慣](habit.md) — オルタスク「習慣・日課」の本格定義
- [ハビットエンジニアリング](habit_engineering.md) — 習慣を扱う方法論

## 引用しておきたい箇所
- 「習慣とは **定期的に行う、定着した事柄**」
- 「習慣の意義は **行動の最大化**」
- 「**重要なのは行動の質ではなく行動の回数**」
- 「**生活の過ごし方が慌ただしく変化する人は定着しづらい**」
- 「**微妙な習慣をさっさと扱うフットワークは、ハビットエンジニアリングの成否を分ける鍵**」
