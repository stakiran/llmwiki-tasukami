# 「タスク管理を眺める」

本書のイントロにあたる章。タスク管理という営みを「眺める」という姿勢で導入し、土台となる概念を一気に提示する。

## 要約

- **タスク管理の本質は「人それぞれ」**。万人に効くやり方は存在せず、自分に合うものを自分で [模索](exploration.md) する他ない。魔法のツールも、AI も、秘書・執事も、現時点では模索を肩代わりしてくれない。
- 模索をゼロからやるのは現実的でないので、本書は「眺めるためのカタログ」として機能する。読者は店頭で物色する感覚で取捨選択する。
- タスク管理は [3P 分類](3p.md) で分けられる: **Project**（チームの仕事）、**Partner**（夫婦・親子）、**Personal**（個人）。本書は主に Personal を扱う。
- タスク管理には [狭義と広義](narrow_broad.md) がある。狭義はタスクを管理することそのもの、広義は目標・環境・精神・調子などタスクの周辺要素まで含めて捉える視点。ブラック企業や睡眠不足下でタスク管理を頑張るのは愚行で、こういう状況では広義の目線が要る。
- [タスク](task.md) の定義は文脈次第で複数あり得る。チームや個人にとって都合のよい定義を採用すべき。定義を持つことで「タスクではないもの」を排除でき、秩序を保てる。
- タスクのように見えてタスクではないものを [オルタスク](altask.md) と呼ぶ。予定、入れ物（フォルダ・タグ）、ビジョン、メモ、習慣などが含まれる。タスクとは異なる扱い方が必要。
- [管理](mca.md) には MCA（Management / Control / Administration）の 3 ニュアンスがある。世間の「●●管理」の多くは Control や Administration だが、タスク管理は **Management**（うまくやる、総合格闘技）。
- [タスク管理](task_management.md) には 2 つの [公理](axioms.md) がある: ① ツールの公理（頭の中だけで完結するものはタスク管理ではない）、② 解釈の公理（何がタスクかは人が決める）。
- 公理から導かれる結論: **タスク管理はスキル**。ツールを使いこなし、意思決定を継続的に下す力が必要で、鍛える対象である。
