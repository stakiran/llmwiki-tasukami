# 「モットー」

[オルタスク](altask.md) のひとつ「ビジョン、理念、標語、心がけ」を本格定義する章。

## 要約

- **[モットー](motto.md)** = 心がけるもの全般（標語、座右の銘、格言、人生哲学、抱負、次から気をつけること、KPT で導いた次やること等）
- モットーの本質的な難しさは **応用** — 然るべき時に思い出し、行動できること
- これを **格言問題** と呼ぶ。タスク管理は「タスクを目に入れて行動」だが、モットーは応用ができないと使えない
- タスク管理の才能には 2 種類: [アスリートとアプライア](athlete_applier.md)
- モットーには **エモットー（External）** と **イモットー（Internal）** があり、行き来して洗練させる

## 章で導入される主な概念
- [モットー](motto.md) — オルタスク「ビジョン・理念・標語・心がけ」
- [アスリートとアプライア](athlete_applier.md) — タスク管理の 2 才能

## 引用しておきたい箇所
- 「**基礎ができているからできるとは限らない。基礎ができていても、その基礎を思い出せないと意味がない**」（応用について）
- 「アスリートとアプライアは両立しない」
- 「**エモットーとイモットーはどちらも使うのが望ましく、むしろ行き来するのが望ましい**」
- 「エモットーは実はかなり漂白されている」
