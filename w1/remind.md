# リマインド (Remind)

指定タイミングに指定要件を思い出させる仕組み。事前にタイミングと要件を仕込んでおくと、そのタイミングで知らされる。

リマインドを行う手段を **リマインダー（Reminder）** と呼ぶ。個々のリマインド設定もリマインダーと呼ぶ。

## メリット

> 予定を確実かつ手間なく思い出せる

| 方法 | 性質 |
| --- | --- |
| 何もしない | 忘れる |
| 常に意識する | 忘れにくいが疲れる |
| 人に頼る | 力が必要、何でも頼めない |
| **リマインド** | 忘れても思い出せる、疲れない、設定は自分で行う |

設定しておけば **意識を捨ててセーブできる**。一度設定したら忘れていい（思い出してくれる）のが大きい。

> リマインダーは認知資源という貴重なリソースを節約する節約術

## 限界

リマインドできるのは「思い出すためのヒントを提示するところまで」。実際の行動は自分次第。

ただし **強度を上げる** ことはできる。

## 強度

弱 → 強の順:

1. 画面上に「14 時」のメッセージを出す
2. 「14 時から会議」のメッセージ
3. 「14 時から昇格面接」のメッセージ（具体的）
4. 3 + アラーム音
5. 4 を 13:50 と 13:30 の二回セット

強度と仕込み手間はトレードオフ。

## リマインダーの種類

### アクティブリマインド
仕組みの方から訴えてくるリマインド。

- 目覚まし時計（音）
- 携帯電話アラーム機能（音、振動、要件指定可）
- 録音目覚まし（自分の声をアラーム音に。アラーム音だと無視してしまう人に効く）
- ポップアップメッセージや通知ダイアログ（視覚）

五感のうち嗅覚・味覚を使ったリマインドはまだ無い（とっさに作用させづらいため、できても遅効的になる）。

### トラップリマインド
[動線](dousen.md) 上にタスク（を想起する物）を仕掛けておく。確実性はアクティブほどではない。

例:

- 玄関ドアにゴミ袋（出社時に目に入る）
- 靴に紙を入れる（履いたときに足裏で気付く、これは触覚で訴えるアクティブとも言える）
- 普段使う TODO リストに行を追加（ハブをトラップ先にする）
- カレンダーに予定として追加（実行確度の高いツールに仕込む）

仕掛け方を工夫すればアクティブ化できる。

### ロケーションリマインド
特定範囲に入ったら / 出たらリマインド。iPhone のリマインダー機能で利用可能。

- 会社を出たら / 自宅に近づいたら → 定期タスクの忘れ防止
- 普段行かないデパートに近づいたら → そこで買いたかったものを思い出せる

頭から要件を外に出して忘れられる利点が大きい（10 場所分を頭の片隅で意識し続けるのと、リマインダー化して全部忘れるのとで認知負荷が天地）。

### ヒューマンリマインド
人に頼る:

- 秘書にスケジュール管理を任せる
- 部下に「会議忘れてそうなら声掛けて」
- 同居人と予定を共有して忘れていたら教えてもらう
- 友達にモーニングコール

メリット: 仕込みがかんたん（ツールに不慣れな人向き）。デメリット: 融通利かない、不確実、要件を相手に教える必要があるため機密性高いものに使えない。

著者の余談: 推しが声でコールするとか、見ず知らずの誰かのリマインドが漂う世界でボランティア的に届ける、など人の介入や声には独特の魅力があり、もっと活用される仕組みがあれば流行りそう。

### 雰囲気リマインド (Atmosphere Remind)
環境の力を借りる。学校の移動教室や体育の例 — 教室・クラスメイト・先生による雰囲気が「次は移動」と圧力をかけてくる。

集団生活の中で「次に皆が参加するべき予定」として制約されるため意外と強度が高い。デメリット: 個人的なリマインドには応えられない。

### 余談: なぜ出社するのか
パンデミック以降リモートワークが可能と判明したのに出社回帰が目立つ理由 = **従業員のタスク管理能力が乏しいから**。

タスク管理が十分行えるなら自律性とスキルでリモートでも回せるが、これは身につけるのが難しい（自然には身につかない、学術体系もない、個人差大きい）。

現代ビジネスの「多忙シチュエーションで瞬発的に行動」「密に連携」のあり方は雰囲気リマインドに依存している（雑談中に会議が始まる等、[専有](property_management.md) と地続き）。

> 多忙縛りなやり方で成果を出せている、かつ、タスク管理やリマインダーを使いこなすスキルと自律性はおそらくない の双方が備わっていると、リモート移行の動機がない

## リマインダーをつくる戦略

### オンデマンド戦略
気付いたらすぐ設定する。[スプリンター](sprinter.md) そのもの。リマインダーをすぐ設定できるよう手元に持ち、操作に手慣れている必要がある。

著者は「2 週間後に面談」と言われたら、その場ですぐ「面談当日」と「3 日前のチェックポイント」のリマインダーを 1 分以内で仕込める。手先の早さは妥協せず追求。

### 定期戦略
定期的にリマインダーを仕込む機会を設ける。

著者の例: 平日朝一に「カレンダーの今日の予定を見て、それぞれ 10 分前にリマインダーを入れろ」タスクを設定。会社 iPhone のアラームを使い、選択肢から選んでオンにするだけ。要件は書かない（音を聞けば思い出せる、思い出せなければカレンダーを見る）。

肝心なのは「リマインダーを設定するタスク」を定期的に仕込むこと。

### 併用が便利
- 定期戦略で今日の予定を取りこぼさない
- 離れた予定や仕掛けたい用件はオンデマンド

オンデマンドが苦手なら週一の「リマインダー設定会」もアリ。

> リマインダーを仕込む機会を意識的に確保する
