# 「探索的タスク管理（ヒント集）」

chap5 の 4 つ目。前章 [exploratory](source_exploratory.md) の part5（ヒント集）が分量上の都合で分離された章。Zenn の 50000 文字制限のため。

## 要約

- まずは [LTM](ltm.md) のテクニックを使う
- **[スクイーズアウト](etm_tips.md#スクイーズアウト)** = 現状の自分で 100% ひねり出す。有限化（中断）の目安、意思決定を加速
- **[ブロックライティング](block_writing.md)** — トピックの育て方の主要技法
    - **CBS**（Consideration Breakdown Structure）= シンプル版
    - **サマリー駆動** = サマリーをつくって仕切り直す
    - **ABCD ブロック** + Summary + 検討 + 用語 + 参考情報 + 文脈
    - **レター** で種類を明示。カスタマイズ可
- **個人と協調の線引き**
    - **トピックは個人でつくる** → 見てもらう（直接修正しない）
    - **パーソナルスペース**（4 レイヤー: 会場/トピック/ブロック/行）
    - **パブリックスペース**で意思決定
    - **表札**（Scrapbox アイコン記法 / 絵文字）
- **個と和のバランス**
    - **興味ドリブン** — 自分の興味関心で動く（義務ドリブンではない）
    - **興味を引く** — 強要せず誘導
    - **スルーを許容する**
    - **オタク注意報**（捲し立て）— 専用トピックで対処
    - **明示的な否定** — ウェイトの解消
    - **役割を工夫する** — ロールプレイ（RP）、動的な役割
- **便利なトピック**
    - **日付トピック** — 共同日記
    - **メンショントピック / ポストトピック** — 通知の代替
    - **サルベージトピック** — 検索 + コメントでまとめ
    - **ステーショントピック** — リンク集の前線基地

## 章で導入される主な概念
- [ETM のヒント集](etm_tips.md) — スクイーズアウト / パーソナルスペース / 個と和のバランス / 便利なトピック
- [ブロックライティング](block_writing.md) — CBS / サマリー駆動 / ブロックライティング本体

## 引用しておきたい箇所
- 「**スクイーズアウトにより意思決定が加速します**」
- 「次のサイクルを開始する際は、なるべく前回のサイクルのサマリー **だけを** インプットにしてください」
- 「**ノートから離れた時点では LTM は終わります**」（LTM 章からの再強調）
- 「**興味ドリブンでなければモチベーションや精神が保ちません**」
- 「**リアクションが必ず来ることを前提にしてはいけない**」
- 「**1日1回届く、でもまだ多いと感じます**」（メンション・ポストトピック）
- 「ステーション ≒ リンク集」
