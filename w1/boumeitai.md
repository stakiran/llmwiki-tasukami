# 忘迷怠（ぼうめいたい）

**忘れる・迷う・怠ける** の総称。タスク管理がカバーする 3 観点のうち、**確実性** にフォーカスする概念。

「確実にこなしたい」というニーズを扱う。

文脈により、忘・迷・怠そのものを指す場合と、その防止や軽減を指す場合がある。

## なぜタスクを残すのか

タスクを書き残しておけば:

- 忘れない（正確には、忘れても見れば思い出せる）
- 何をやるか迷わない
- リズムに乗れば怠けず消化できる

ただし 100% うまくはいかない。「タスクは見たし、認識もしたし、このあと行動するつもりだった」のに実際しなかった、というサボタージュもある。それでも目に見えないよりはるかにマシ。

## TODO リストでは足りない

「TODO リスト」は有名だが「役に立たない」ことでも有名。タスクを並べただけでは、100 個並んでいても全部消化できない。

だからタスク管理ツールが存在する。優先順位、カテゴリー、状態（未着手・進行中・完了）、締切、開始日といった属性をタスクに持たせ、フィルタや並び替えで見せ方を工夫することで、ようやく機能する。

ただしツールを機能させるには、日頃からタスクと属性を細かく更新する必要があり、これ自体が [トレメント](trement.md) になる。

## 質を上げる鍵: 動線

> ツール上に存在するタスクを見れば、ある程度は防げる

ただし「見る」が成立するためには、ツールを [動線](dousen.md) に組み込む必要がある。これが [RSAF サイクル](rsaf.md)（Read → Select → Act → Feedback）を回す前提条件。

動線以外の方法もある:

- カレンダーは多くの人にとってすでに動線になっている
- パートナーや家族のフォロー
- 物理的に物を置く（風呂掃除セットを浴室前に置く等）

現実的には複数の方法を組み合わせるのがベスト。
