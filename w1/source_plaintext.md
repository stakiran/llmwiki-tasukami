# 「プレーンテキストによるタスク管理」

chap5 の 1 つ目。アナログのような「自由自在さ」をデジタルで実現するため、プレーンテキストとエディタによるタスク管理（PTTM）を扱う章。

## 要約

- **[PTTM](pttm.md)**（Plain Text based Task Management）= プレーンテキストとエディタを使ったタスク管理
- 目的は **自由自在** の獲得。アナログの紙とペンの感覚をデジタルで実現
- PTTM ≒ **タスクリストの運用 + [属性](attributes.md) の工夫**
- 前提知識は [プレーンテキスト](plain_text.md) と 5 つの道具（IME / エディタ / 形式記法 / 情報管理 / ヘルパー）
- 設計原則 6 つ: 管理範囲 / 動線化 / **管理的安全性**（誰にも見せない）/ 管理対象 / 振り返り / **3 秒の壁**
- PTTM では扱わないもの: [オルタスク](altask.md) / [ルーチンタスク](routine_task.md)
- 設計概念: デイリータスクリスト / **曖昧な日付感** / フォーマット統一 / 半角スペース統一

## 章で導入される主な概念
- [プレーンテキスト](plain_text.md) — 媒体・道具の前提知識
- [PTTM](pttm.md) — プレーンテキストによるタスク管理本体

## 引用しておきたい箇所
- 「**アナログの専売特許だった「自由自在」を、メリットの多いデジタルにおいても実現できる**」
- 「プレーンテキストの操作は **カーソル操作をいかに上手くやれるか** にかかっている」
- 「**情報単位間（たとえばファイル間）を行き来するコストは高い**」
- 「**動線化が定着するかどうかが PTTM の成功の分かれ目**」
- 「個人タスク管理が形骸化する主因の一つは **「恥ずかしいから」**」（管理的安全性の動機）
- 「3 秒以内にメモを取れる――これができるくらい素早くメモが取れるなら、あなたの PTTM は手に馴染んでいる」
- 「**自分に合ったやり方はあなたにしかわからず、自分で模索していく**」
- 「フォーマット統一は PTTM を続けたいなら絶対にやっておきたい投資」
