# Robot

「このとおりに動けば一日が上手くいく」というレベルで詳細な [ルーチンタスク](routine_task.md) リストをつくり、それに従って行動する [戦略](strategies.md)。繰り返しの力でまわす系。

## 概要

人の生活の大半はルーチンタスクに帰着できる。Robot のタスクリストは多数のルーチンタスクが並ぶ。最初から都合のよいリストはつくれず、記録や仮説で少しずつ明らかにする。**定着まで一ヶ月どころではない**。

例（著者の場合）:

```
-5:00 🔁（毎日）睡眠おわり、起床
-5:04 🔁（毎日）トイレとカーテンと顔洗い
-5:07 🔁（毎日）お湯を沸かす、マットレスを片付ける
-5:10 🔁（毎日）コーヒーとパンを持ってデスクに
-5:40 🔁（毎日）情報のキャッチアップ、コミュニティへの返信
-5:50 🔁（平日毎日）勤務準備
-5:50 🔁（火・木）可燃ごみ ごみ捨て
-5:55 🔁（平日毎日）リモート勤務勤務開始
...
```

決まったリズムで動く様はロボットのよう。**著者にとっては自然**。他の人も自覚がないだけで自分なりのリズムがある。

## ツール

ルーチンタスクを扱えるツールが必須:

- **タスクシュート** — 国内では火付け役
- **Todoist** — シンプルでも扱える
- カレンダー系では力不足（一日何十タスクの操作に耐えない）
- 著者は Monolith の延長で **Tritask** や **todaros** を自製

## 応用

### どこまで固めるか
時間帯（朝〜寝るまで全部か、勤務中だけか etc）、タスク（家事だけか、習慣全般か etc）、実行の記録（開始終了まで or 終了だけ or 終わったらいいか etc）、順序性（厳密上から or 飛ばしていいか etc）。生活全域への適用は趣味レベル。少しずつから。

### 強いロボット / 緩いロボット

| 種類 | 性質 |
| --- | --- |
| **強いロボット** | 厳しく従う。安定感抜群だが融通が利かず、乱れるとストレス。20 人に 1 人もいないかも |
| **緩いロボット** | 従うかゆるく決める。「今日ごみ捨ての日だけどだるいから次でいい」もアリ。安定感はないが融通が利く、忘れない |

「だったら Robot しなくていいのでは」と思いがちだが、タスクリストが並んでいること自体に意味がある。

> 生活において必要なルーチンタスクのすべてが集まっており、しかも適切な頻度で出現するという安心感

緩いロボットでも、すべて集めることと適切な頻度のチューニングは自分で行う必要がある。

### リズムの種類
- **タイミング** — いつ実施するか
- **頻度** — n 日に 1 回。1 日に複数回ならタスクを複製
- **キャパシティ** — どれだけ詰め込めるか。多すぎると形骸化（怠ける → 忘れる）。著者のキャッチアップは n=3、それ以上苦しい

リズムは仮説検証で微調整。

> こういう微調整ムーブを息するように行えるかどうかが、Robot 定着の分水嶺になる

ダメなら他戦略へ、または諦める。

## メリット
- 自分のリズムに則った安定感
- すべてのタスクが適切頻度で出現する安心感

## デメリット
- ハードルが非常に高い、人を選ぶ、定着の道のりも長い
- 一生微調整の世界観のしんどさ（著者も今は部分採用のみ）

## 適性
- **几帳面な人**
- **仕事と家事育児を両立する忙しいビジネスパーソン** — 忙しさで生活リズムが固定化されるため。ただし [盤外戦](bangaisen.md) で時間とアクセス手段を確保する必要あり
- **多動・ずぼらな人には絶望的に向かない** — トレメントが最も求められる戦略
