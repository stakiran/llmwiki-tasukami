# パートナータスク管理

夫婦や親子など、恒常的に一緒に暮らしている親密な少人数の家族間で使われるタスク管理。[3P 分類](3p.md) の Partner に対応する。著者独自の呼称。

「2 人組」のニュアンスで使われるが、3 人以上のあり方（子供込み、ポリアモリー等）にも当てはまり得る。シェアメイトでも成立する可能性。

扱うタスクは **日常生活全般** — 家事、育児、引っ越しや旅行といった家庭イベントの分担。同じ会社に勤めているなど同一仕事の管理は [プロジェクトタスク管理](project_task_management.md) の範疇で、本章の対象外。

## 成立の難しさ: [ファミリア・デバフ](familiar_debuff.md)

タスク管理は面倒くさく、スキルとリテラシーが要る。一方でパートナー相手だと怠惰が許されてしまうため、面倒な諸活動がサボられて成立しづらい。

## ツールの変遷

| 段階 | 手段 | 性質 |
| --- | --- | --- |
| 1 | 口頭・電話 | コミュニケーションのみでタスク管理ではない、言った言わなかったが多発 |
| 2 | ポケベル・メール | 電話と本質的には同じ。IT 音痴は使えない |
| 3 | LINE / スマホ | メッセージ高速化、スタンプ。だが非同期的に使える人は少なく、結局口頭の域 |
| 4 | Slack / Discord | **夫婦 Slack** / **カップル Slack** / **ひとり Slack** とも呼ばれた。チャンネルで用途別の部屋、検索、連携、リマインダー |
| 5 | 専用アプリ | TimeTree（予定共有）、Cross（夫婦 ToDo 共有）、家事の名は / Yieto（家事見える化） |

LINE が「自分のペースで送ってもいい／空いた時に返事すればいい」非同期的手段として使える人は意外と少ない。現代人は SNS の影響で通知に反応する体質。非同期的な考え方は（[専有の管理](property_management.md) でも見たとおり）高度なもの。

専用アプリが本質を突いている部分:

- カレンダーで俯瞰できる（[Calendar View](3c.md)）
- 個別タスクにアクセスできる（[Chat per Task](3c.md) 寄り）
- 個別タスク上でチャットができる（話題が混線しない）
- タスク管理を知らない人でも比較的使いやすい

## 本質: [3C](3c.md)

- **Chat per Task** — タスクごとのチャット
- **Calendar View** — カレンダーによる俯瞰
- **Concept** — 細かいコンセプトで差別化

## 鍵は [盤外戦](bangaisen.md)

タスク管理よりも盤外戦の方が重要。意識的に盤外戦の時間（最低でも週 1 時間）を捻出する。

実行できるかどうかは [トーカビリティ](talkability.md)（対話できる素養）にかかる。

## プロジェクトタスク管理への示唆

[プロジェクトタスク管理](project_task_management.md) では、パートナータスク管理が **タスク管理とメッセージのやりとりを上手く融合** している点を評価している。ビジネスではなく私生活としてかんたんに使えるバランスも確保しており、[Alternate](alternate.md) における注視機能の洗練の好例。著者は「この発想を膨らませればプロジェクトタスク管理ももう一皮剥ける」と見ており、後の章（高度なタスク管理）で再度取り上げられる予定。
