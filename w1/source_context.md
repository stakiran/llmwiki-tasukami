# 「コンテキスト」

[調動脈](choudoumyaku.md) の章 2 で「重要かつ奥深い概念」として持ち越されていた **文脈（Context）** の本格定義。chap4 の 3 つ目。

## 要約

- [コンテキスト](context.md) は文脈、ある場面や対象に紐づいた周辺状況や付随情報
- 動的（状況）と静的（属性）がある
- タスク管理におけるコンテキストは **5 種類**: タスク / セルフ / ツール / トリガー / [ロード](load_context.md)
- 人に対しては試行錯誤ができないので一発で決めるためにコンテキストが要る（**文脈理解猶予** が要る）
- ロードコンテキストの切り替えが [コンテキストスイッチング](load_context.md) で、これは疲れる営み

## 章で導入される主な概念
- [コンテキスト](context.md) — 主題と 5 種類の概要
- [ロードコンテキスト](load_context.md) — コンテキストスイッチング、スプリント/マラソン、土地勘

## 引用しておきたい箇所
- 「**人に対しては試行錯誤ができない**」
- 「**合うのなら嫌いでも使った方がいいですし、合わないのなら好きでも使わない方がいい**」（ツールコンテキスト）
- 「**コンテキストスイッチングはとても疲れます**」
- 「**主流に惑わされず自分のタイプを判断してほしい**」（スプリント vs マラソン）
