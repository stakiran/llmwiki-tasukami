# 「探索的タスク管理」

chap5 の 3 つ目。基準に頼らず探索することで視界を広げるタスク管理（ETM）の基礎編。前章 [文芸的タスク管理](source_literate.md) を前提とする。

## 要約

- **[ETM](etm.md)**（Exploratory Task Management）= 基準に頼らず好き勝手に動いて視界を広げていくタスク管理
- **No ABCD**（Assign / Ball / Consensus / Deadline）= 探索的の核
- メリットは **自由と楽しさ**、目的は **視界を広げる**、適用は **Unknown Unknown**
- 従来の **指向的アプローチ** と対比。両者は **ハイブリッド**（まず探索 → 大体見えたら指向）で使う
- 情報単位 4 種: **ゴール / カード / トピック / ブロック**
- 育て方 3 操作: **広げる / 詰める / 導く**（要約 vs 蒸留 = **ダシにする** = ブレイクスルー）
- ETM ≒ トピックを用いた知的生産（KJ 法など既存手法のリファレンス）
- 管理は 5 つ。詳細は [ETM の 4 つの管理](etm_management.md) 参照
    - 1 ロード（一桁の原則 / ロードキャパシティの可視化）
    - 2 ターゲット（**3T モデル** = Topic / Target / Task、**当事者**、**単一当事者の原則**、**ブースター**）
    - 3 リソース（**管理 2.0** = 状態の計測。ミニマイザー vs マキシマイザー、ネガティブ因子重視）
    - 4 コンテキスト（コンテキストメンテナンス vs コンテキスト駆動。同時不可）
    - 5 タスク（LTM 流）
- ワークフロー 3 ステップ: セットアップ（**探索期間**、**探索時間 ≧3 時間**、**ミュートデイ / ミュートウィーク**）→ ETM → ゴール判定
- **動的な ETM**: 探索モードと指向モードの切り替え

## 章で導入される主な概念
- [ETM](etm.md) — 探索的タスク管理本体（情報単位、育て方、ワークフロー）
- [ETM の 4 つの管理](etm_management.md) — ロード / ターゲット / リソース / コンテキスト

## 引用しておきたい箇所
- 「**No ABCD** で行います」
- 「楽しさは重要。**私たちは怠け者**」
- 「**ETM とはブレイクスルーを味わうための創造的な営み**」
- 「**ETM はあくまで行動する直前の、準備や整理の部分までを済ませるもの**」
- 「結果を管理するのではなく、**結果を生み出すもとである私たちの状態を計測しましょう**」（管理 2.0）
- 「ETM を上手くやれるかどうかは、**コンテキストスイッチングの消耗をいかに抑えるかにかかっている**」
- 「**探索期間という聖域は絶対に侵攻させてはなりません**」
- 「**リソースの管理には結局痛みを伴う行動が求められます**」
