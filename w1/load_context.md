# ロードコンテキスト (Load Context)

そのタスクを行うとき頭に入れておく文脈。仕事を切り替えたり休んでから再開する際、その仕事の文脈を頭に入れる必要がある。これを切り替える営みが **コンテキストスイッチング**。

[コンテキスト](context.md) の 5 種類のうち、唯一の動的なコンテキスト。

## コンテキストスイッチングはとても疲れる

> コンテキストスイッチングはとても疲れます

例: ある管理職が幹部報告、定例会議、5 人の部下との 1on1、家庭でのパートナー・子供とのやりとりを抱える → 6 + 部下分の異なるロードコンテキストがある。

**重要なのに疲れる** というシビアな営み。

> 切り替えの腰が上がるようになると、行動力が上がります

体力やバイタリティの問題というより、**スイッチングの疲労** という邪魔者を一つ薄められる。

## コンテキストスイッチング・モデル

著者考案の直感的モデル。脳科学的根拠ではない。

- ロードコンテキスト A から B にスイッチングする
- それぞれ広がりを持つ **領域** で、その間を歩いて移動する
- 中央に **ニュートラル**（回復地帯）があり、経由する

### 4 つの原則

| 原則 | 内容 |
| --- | --- |
| **地道の原則** | 移動には時間がかかる |
| **回復の原則** | ニュートラルを経由するべき |
| **面の原則** | 領域なので広がりや重なりがある（ゼロイチではない） |
| **等価交換の原則** | 領域の奥底に行けば深く浸れるが、行くにも戻るにも時間がかかる |

## 戦略

### 休憩の時間をしっかり取る
ニュートラル経由のため。強度（著者の体感）:

```
睡眠 >> 昼寝 > 瞑想・ぼーっと >> ネットサーフィン・談話 > 家事・散歩・手慣れた作業
```

ネットサーフィンは認知資源を、散歩は体力を消耗するので **使い分けが大事**。1 種類に頼ると別の資源が枯渇する。

> 1時間仕事X → 1時間ネットサーフィン → 1時間仕事Y → 1時間散歩 のように同じ時間だけ休憩を差し込んでも全然問題ない

スイッチングが不十分のままだと仕事が雑になり進行も遅くなる。

### スプリント vs マラソン

| スタイル | 性質 |
| --- | --- |
| **スプリント** | オンオフをしっかり区別。ロード領域の奥底に一気に突っ込み、しばらく滞在後一気に駆け抜ける |
| **マラソン** | だらだらとながら作業しつつ複数こなす。中途半端な過ごし方。常に歩いている |

目安: 30 分〜1 時間の散歩を毎日続けるとして、散歩中に他のことを行いたい / 行ってしまう → マラソン派。

> 現代ではマラソンが主流（スマホ/SNS、長時間拘束、マルチタスク要求）

著者は昔マラソン派、現在スプリント派（スマホを持たないなど現代潮流に抗う行動）。

> 主流に惑わされず自分のタイプを判断してほしい

### バッファを取る
スイッチングは移動だから時間がかかる。前後に **5〜10 分でも余分に取る**。会議連続詰め込みも数分の休憩があるだけで違う。負荷は重なると効く。

詳細は [バッファとスラック](buffer_slack.md)。

### 土地勘を養う

#### 位置関係（ニア）
タスク同士が文脈的に似ているか。観点が多いほど近い:

- 内容感（カテゴリー）
- 締切感（今日が同じならスイッチングが楽）
- 資料感（人に伝える必要性）
- 審査感（チェックの必要性）
- 契機感（同じトリガーコンテキストか）

最低 1 つ、できれば複数の **ニア（near）** を探したい。把握できないなら書く・プロットする（**Miro** など無限キャンバス）。

#### 歩きづらさ
タスクの性質と実行者の能力で決まる。

- **歩きづらいタスク** — 一日の中で実行できるタイミングが限られる。例: 著者の小説執筆は起床後のフレッシュな頭で数時間。スポーツの本番、アイデア出し
- **歩きやすいタスク** — 一日中どこでもできる。始めれば進む。例: 慣れた作業

歩きづらさを把握すると **取り組み方の設計** ができる。歩きやすいタスクは「自律的に始めるのが難しい」ので環境や集団の力に頼る（カフェで仕事、たくさん詰め込む）。

ゲームのマッピングと同じ。一度つくれば以後は最適行動が取りやすい。
