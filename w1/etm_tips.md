# ETM のヒント集

[ETM](etm.md) を実践する際に役立つやり方や考え方。ゼロから試行錯誤するよりも近道できる。あくまでヒントであって正解ではない。

> [文芸的タスク管理](ltm.md) も参考に。ETM は LTM の上に探索的アプローチを重ねたものなので、LTM の諸テクニックも使える

## スクイーズアウト

> 現状の自分で 100% ひねり出すこと

**Squeeze Out**。ETM の有限化（中断）の目安として使える。

ETM では「見えてきたら」が合図だが、いつも見えてくるとは限らない。**正解はないため自分の意思決定で中断する有限化** が必要。

> 今の自分で出し切れるのはこんなところかな～
> これ以上は出なくなったんでいったんここまで
> 飽きたんでこれくらいで

著者は「胸を張れるか？」との自問自答を使う。

ベストエフォートとの違い:

| | 内容 |
| --- | --- |
| **ベストエフォート** | 「最大限頑張ります」と言うだけで情報量なし |
| **スクイーズアウト** | 出し切ったかどうかを評価する。主観だが端的な目安 |

> スクイーズアウトにより意思決定が加速する

出し切っているなら納得感が高く、上手く行かなくてもインプット不足や見落としを疑える。これ以上自分で何とかすることはできないと胸を張れて、意思決定が早まる。

## 個人と協調の線引き

複数人で ETM を行う場合、各個人の書き込みを尊重できないと居心地が悪くなり形骸化する。

### トピックは個人でつくる

> 誰か一人で率先してつくり、それを他の人に見てもらう

従来は最初から打ち合わせで話し合いがちだが、ETM では捨てる。**一人でつくってレビューしてもらう** スタイル。

トピックには正解がないので、最初から協調するとこじれる。誰か一人がとりあえず形にして、皆で好き勝手にこねる方が良い。

A さんが書いた T1 が気に入らない B さんは、T1 を直そうとせず **B さんが T2 をつくって A さんに見せる**。A さんも T2 を直接修正せず、自分の T1 を修正する。**個人の領分を尊重しながらお互いに言及し合う**。

[当事者は可能な限り 1 人](etm_management.md#単一当事者の原則) と地続き。

### パーソナルスペース
**個人の領分**。レイヤーは 4 つ。

| レイヤー | 名前 | 内容 |
| --- | --- | --- |
| **4** | **会場** | 会場そのものが個人の領分。ETM では非推奨（メイン会場は 1 つに） |
| **3** | **トピック** | ETM で頻出。一人でつくる原則の単位 |
| **2** | **ブロック** | **最頻出**。1 トピック内に N 人のパーソナルスペース |
| **1** | **行** | 行単位（単一行 / 複数行） |

> どのレイヤーを領分と感じるかはその人次第、その状況次第で変わる

尊重できないとまともに書けず形骸化する。日本はハイコンテキストで「我慢」「無視」になりがちなので、率直な議論をした方が良い。

#### 会場レイヤー
ETM のメインから外れた個人会場。基本的に非推奨。用途は:
- メインに書けないか書いても役に立たない「個人的すぎる」内容
- 個人の情報発信手段（公開 ETM）

メインからリンクが必要。

### パブリックスペース
パーソナルスペースの逆。誰でも入れて誰も領分を主張できない共有地。

> トピックにはパーソナルスペース的なブロックと、パブリックスペース的なブロックが並ぶ

ハイブリッドでパブリックもパーソナルも両立。

意思決定はパブリックスペースで行う。各自が意見を並べて終わりではなく、**暫定的でもいいので確定する**。

### 表札
ブロック / 行レイヤーのパーソナルスペースを示すマーク。

- Scrapbox はアイコン記法（Ctrl + i で自アバター挿入）
- それ以外なら **絵文字**（著者は🐰、辞書登録「うさ」）
- 古典的には `(吉良野すた)` だが面倒で視認性低い

例:
```
🐰
 アイデア出し、くらいの認識
 個人か集団かは関係ない
  ただ集団でやるのは正直無理
   🐴人類には早すぎると思う理由が気になる
   🐰こっちで → [ブレストは人類には早すぎる]

🐴
 オズボーンが考案した会議手法じゃないの？
```

🐰のスペース内に🐴の行レイヤースペースが入れ子になっている例。

## 個と和のバランス

### 興味ドリブン

> 自分の興味関心に基づいて自然に動く

仕事や私生活では **義務ドリブン** が普通だが ETM では望ましくない。ETM は化学反応を期待する **遊び**。

> 整合性が取れるのか?

意外と取れる。重要・面白いトピックは自然と盛り上がるため。寂しいトピックも 100 失敗のうちの一つで OK。

> 何より興味ドリブンでなければモチベーションや精神が保たない

義務ドリブンが従来成立していたのは正解があるため。ETM には正解がない。

### 興味を引く
強要できないので、興味を引く書き方をして誘う。

| 手段 | 内容 |
| --- | --- |
| 動線上で紹介 | [日付トピック](#日付トピック) で雑談的に取り上げる |
| アイキャッチとエキサイトメント | [LTM のテクニック](ltm.md#アイキャッチとエキサイトメント) を流用 |
| キャッチーなトピック名 | 平易・わかりやすく |
| 関連トピック T1 上で紹介 | T2 が T1 の問題解決として自然に登場 |
| ギブ重視 | 他のメンバーを支援する。情報は出す人のもとに集まる |

ETM の他メンバーは仲間であると同時にアーリーな顧客。

### スルーを許容する

> 無理してリアクションを取ったり返信を書いたりする必要はない

ETM は情報を並べる世界で、お礼や挨拶などのリアクションは不自然。情報量皆無でノイジー。

> リアクションが必ず来ることを前提にしてはいけない

「よくお礼を言う人」「ほとんど言わない人」が混在するが、個性として許容する。ETM では仲良くなる必要はなく、トピックを書いてゴールをひねりだせれば良い。

### オタク注意報（捲し立て）

> 自由だからといって自分しか興味がないコンテンツをたくさん共有するのは控える

**捲し立て（まくしたて）** = オタクの早口語りのように一気に多数のトピックをつくる行為。例: 自分の嗜好の書籍・音楽・映像を 1-作品 1-トピックで一気に 20 個。

ノイジーなので避ける。会場の更新順ビューが乱れる。どうしても捲し立てたい場合は **専用トピック**（「映画好きの俺がおすすめをひたすら紹介するページ」）をつくってそこでやる。

ETM の会場は展示場ではない。

参考: [WOM(Write Only Member) - 橋本商会](https://scrapbox.io/shokai/WOM(Write_Only_Member))

### 明示的な否定（ウェイトの解消）
**ウェイト（Wait）** = メンバー全員のコメントを聞かねばならない場面。No ABCD では本来発生しないはずだが、現実にはある。

> ETM ではウェイトは可能な限りゼロに近づけるべき

慢性的に「来ない」場合はメンバー不適。たまの遅延は **ネガティブな反応がしづらい** ため。

**明示的な否定（Explicit Negative）** = 「いいえ」「だるいんでやめときます」「興味ないんでどうでもいいです」を明示的に書く。

日本の文化だと否定はしづらいが、宙ぶらりんは催促者の負担になるので **明示的に否定** する。問題が生じるなら別途トピックで議論する。

> 怠惰は許容しない

催促者の負担を生むため。否定のしづらさが主因なので、明示的な否定で打開する。

### 役割を工夫する
**役割** = ある得意領域とそれに基づく行動。通常メンバーには複数の役割がつく。

役割の例:

| 役割 | 内容 |
| --- | --- |
| **ブレスター** | 思いつき・アイデアの発散が得意 |
| **サマライザー / ディスティラー** | 要約 / 蒸留が得意 |
| **リンター** | 文法的・慣習的な書き方の修正 |
| **リンカー** | あるトピックに関係する別トピックを提示 |
| **意思決定者** | 最終的な意思決定。通常一人 |
| **スティッカー** | 厳しい意見をズバッと書く |
| **トリックスター** | 他メンバーが真似できない独特なことを書く |
| **リマインドマン** | 反応の遅い人に催促 |

> 役割のとおりに行動しろと言っているのではなく、その役割が示すアクションが得意であるか、よく行うか、との意味

#### 動的な役割
ETM では役割は動的。状況や気分に応じて変わるのが自然。

#### ロールプレイ (RP)
**Role Play**。「じゃあ今からブレスターとして書き込んでみます」のように使う役割を宣言してそのとおり行動する。

進展がないときややる気が起きないときに有効。やりすぎるとウザいので頻度・タイミングを調整する。

役割は **「～～役」と書かれた帽子** のメタファー。誰でも被れる。被れば見えるので「A さんは今ブレスターだな」とわかる。適性診断（[ストレングスファインダー](https://www.amazon.co.jp/dp/B07W6RX9N7) / [MBTI](https://www.16personalities.com/ja/%E6%80%A7%E6%A0%BC%E8%A8%BA%E6%96%AD%E3%83%86%E3%82%B9%E3%83%88)）とは違い、苦手だけど一時的にやる、ランダム割り当ても可能。

## 便利なトピック

つくると便利なトピック。必要に応じて生み出してもよい。

> トピックの使い方を制する者は ETM を制する

### 日付トピック
[LTM の日付ノート](ltm.md#日付ノート) を複数人 ETM に応用。

各メンバーが [パーソナルスペース](#パーソナルスペース) をつくってつぶやき・日記を並べる。誰が何をしているか、雑談・相談・情報共有が弾みやすい。

単位は **日単位** がおすすめ。書き込み量が少ないなら週単位でも OK。**ノートの作成は誰かが毎回行う**（自動化推奨）。期待して様子見すると誰もつくらず形骸化する。

### メンショントピック / ポストトピック
ETM では声掛け・メンション・通知が使えないが、見てほしい場面はある。そのためのトピック。

| | 内容 | 仕組み |
| --- | --- | --- |
| **メンショントピック** | ノート名で直接呼びかけ | `@Aさん アイデアX見てほしいです` というノート名 |
| **ポストトピック** | 受信箱的なノート | `inbox @A` ノート、A さんに用事のある人がそこに書く |

> 両方使うとノイジーなので片方だけ

著者推奨は **メンショントピック**。ポストトピックは持ち主の負担が大きい（チラシのような感覚）。

> 1 日 1 回届く、でもまだ多い

そもそもこれに頼るのは ETM に反する。基本的に一度も使わないに越したことはない。多用が必要ならセットアップが間違っている可能性。

### サルベージトピック

> ヒントは散らばっているが正解が示されていない状態であることが多い

**Salvage** = 救出・復旧。トピックの海から特定話題を復旧する。

具体的には **検索結果をコメントつきでまとめる**。

```
リモートワークのサルベージ
 [Remotty]
  ソニックガーデンのバーチャルオフィスツール。2分ごとに自席のカメラ画像を反映
  🐰さんが [ソーシャルプレゼンス] と一般化してる
 [デスク爆弾]
  >職場などで自分のデスクに前触れもなく近づいてきて延々と雑談や仕事の相談をされること
  Z世代のワード
 [ラセングルのリモートワーク]
  FGOをつくってる会社のリモワ工夫事例
```

特に **個人 ETM** で重宝。複数人でも「人によって捉え方が全然違う言葉」（プロジェクト・タスク管理・DX）に有効。

デメリット: 純粋に疲れる、実践者の主観が混ざる。

### ステーショントピック

> 作業デスク的な前線基地

ある話題 T を深堀りしたいときに、T のステーション（前線基地）をつくる。トピック名には必ず「ステーション」をつける（例: 「自転車ステーション」）。

> ステーション ≒ リンク集

検討は別トピックで行い、ステーションからはリンクを張るだけ。リンクを **自分の言葉で束ねる** のがコツ。

```
日記で盛り上がってる日
 [2024/08/21]
 [2024/08/15]
 [2024/08/14]
 [2024/08/11] ここが発端、なぜか自転車の話題が大盛りあがり

自転車ネタのトピック例
 [ママチャリ vs クロスバイク]
 [A-bike city 折りたたみ自転車]
 ...

自転車ネタから出てきた議論やアイデア
 [自転車通勤を認めてほしいです]
 [広い敷地を持つ事業者向けにスポーツバイク導入を提案する]
 ...
```

> 良いステーションの目安: このトピックから辿って読んでいくのが一番近道

LTM の言葉では、ステーションは **リンクばかりが並んだ [トランク](ltm.md#トランク)**。
