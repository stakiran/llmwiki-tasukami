# 「文芸的タスク管理」

chap5 の 2 つ目。文章を読み書きする営みをベースとしたタスク管理（LTM）を扱う章。

## 要約

- **[LTM](ltm.md)**（Literate Task Management）= 文章を書く営みを通じたタスク管理。文脈の尊重がメリット、QoL の向上が目的
- ルーツは **文芸的プログラミング**（クヌース、Jupyter Notebook）
- ノートが必要不可欠。SNS / チャット / 多機能ノートツールでは足りない（軽さとまとまった文章の扱い）
- 推奨ツール: **[ネットワーク型ノートツール](network_note.md)**（Scrapbox / Obsidian）
- スタイルは 3 つだが LTM ≒ ネットワークベース（**トピック指向**、1-話題 1-ノート）
- **ホーム** の 4 種: ホームノート / ホームビュー / 日付ノート / トランク
- LTM の情報は 3 種: メモ / タスク（**タスクビュー** + **タスクノート** の両輪）/ 文脈
- 後で読み返す戦略は **エリアとマーク** の 2 つだけ
- LTM でカバーできない部分は **アンカバード**（既存ツールに頼る）
- 文脈は「ヒント」。書きたいこと / 深堀りネタ / 自問自答
- **ライティングスタック** — 原本を頭の中ではなくノートに置く
- **切り出し** で文章を部品化、**エイリアス** で扱いやすさを高める
- **アイキャッチとエキサイトメント** で継続のモチベを保つ
- 本質はノートテイキング。**見えてきたらさっさと行動する**

## 章で導入される主な概念
- [ネットワーク型ノートツール](network_note.md) — リンクでノートを繋ぐパラダイム。LTM の前提
- [LTM](ltm.md) — 文芸的タスク管理本体

## 引用しておきたい箇所
- 「**文芸的タスク管理は一応タスク管理といいつつも、本質はノートテイキング**」
- 「**ノートから離れた時点で LTM は終わる**」
- 「**ホームとして常に表示し、常に見るノートは一つだけに留めたい**」
- 「**LTM が定着するかどうかは、ライティングスタックの考え方に染まれるかどうかにかかっている**」
- 「**切り出しが文脈をつくる**」
- 「**ノートの読み書きには正解がなく、正解に向けて進めばいいという単純な世界には落とし込めない**」
- 「LTM の意義は、**タスクを決める段階にフォーカスした手法であること**」
