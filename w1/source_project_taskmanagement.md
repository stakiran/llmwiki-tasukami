# 「プロジェクトタスク管理」

[3P 分類](3p.md) のうち Project を端的に整理する章。本書の主題は個人タスク管理だが、プロジェクトタスク管理についても本質を整理しておく。PMBOK のような体系や即効性のあるテクニック紹介はせず、個人タスク管理を体系化する目線でまとめている。

## 要約

- プロジェクトタスク管理の本質は **3A**: [アサイン](assign.md) / [調整](adjust.md) / [俯瞰と注視](alternate.md)
- 俯瞰と注視の実現度合いには [4 つの段階](alternate_model.md) があり、第 3 段階（タスク管理ツールの常用）に **キャズム（深い溝）** が存在する
- 俯瞰用の [ビュー](views.md) には 6 種（リスト / テーブル / ボード / チャート / ネットワーク / センテンス）。当面はテーブルが無難
- プロジェクトタスク管理上に計上されない隠れタスク = [シャドータスク](shadow_task.md)。**シャドーエフェクト** が生産性を蝕む
- 対処は [キャパシティベース](capacity.md)、[バッファとスラック](buffer_slack.md)、ダイレクトシャドーのタスク化
- プロジェクトタスク管理は実は [5 つの管理対象](management_targets.md) を背負っている: [進捗](progress_management.md) / [質](quality_management.md) / [プロセス](process_management.md) / [リソース](resource_management.md) / [専有](property_management.md)
- 専有は [脱拘束](unbinding.md)（非同期コミュニケーション、QWIC、Communication Injection）で軽減する
