# 「動線とリマインド」

chap4 の最初。タスク管理を行動に繋げるための仕組みを扱う章。章 2 で予告された [動線](dousen.md) の本格定義と、新しく [リマインド](remind.md) を扱う。

## 要約

- [動線](dousen.md) と [リマインド](remind.md) は、忘迷怠を減らして適切なタイミングで行動するための仕組み
- 動線は **ハブ（Hub）** と **ジョイント（Joint）** に分けられる
- 動線リマインドの設計（**動線のデザイン**）は **ネグレクテッド** と **ロスト** のバランスが鍵
- リマインダーには様々な種類: アクティブ / トラップ / ロケーション / ヒューマン / 雰囲気
- リマインダーをつくる戦略は **オンデマンド戦略** と **定期戦略** の併用
- [RSAF サイクル](rsaf.md) は動線リマインドをハブで運用するためのあり方

## 章で導入される主な概念

- [リマインド](remind.md)
