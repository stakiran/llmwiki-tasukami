# プロジェクトタスク管理

仕事におけるチームなど、複数人で行うタスク管理。[[3p]] の Project に対応する。

## ツール例
- Trello
- Asana / ClickUp / monday.com
- Redmine / JIRA / GitHub Issues
- Microsoft 365 / Notion

## フレームワーク例
- WBS (Work Breakdown Structure) / ガントチャート
- PERT (Project Evaluation and Review Technique) / CPM (Critical Path Method)
- カンバン
- バーンダウンチャート

## 変遷
- かつてはプロジェクトマネジメント由来のフレームワークが主流。手作業や Excel / スプレッドシートでも運用できた
- 現在は SaaS 型のタスク管理ツールが主流。タスク単位でアサインややり取りができ、リストやボードで全体俯瞰できる
- 近年は **連携・自動化・カスタマイズ** が高度化している

## 本質

大きなタスクの分解、締切と担当者の設定、俯瞰用ビュー、全員が同じ場所を見ること（一元化）。要は皆の方向性と認識を揃えるために、タスクという情報を外に出して原本にする。**ものすごく単純に言えば、ただの見える化**。

## 限界

> 見える化する以上の能力は無い

- 結局口頭やチャットで催促が要る
- 更新が形骸化して、ツールではなく人に聞きまわるあるある
- 高頻度の会議体で補うあるある
- 目の前のタスク消化に専念しがちで、改善・勉強・新規検討といった中長期がおざなり

## 使われる層

主に現場で使われる。中間管理職以上は、扱う情報量も人も多く状況も変わるので、ツールではなく自分の脳内＋ミーティングで何とかしがち。

## 総括

> やると決まったもの（あるいは決めたこと）を確実にこなすために、やることを外部化すること、およびそれをメンテナンスすること
