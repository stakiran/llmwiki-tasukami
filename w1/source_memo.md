# 「メモ」

[オルタスク](altask.md) のひとつ「メモ」を本格定義する章。

## 要約

- **[メモ](memo.md)** = あとで処理することを想定した一時的な記述
- フローは **書く → 読み返す → 行動と処理** の 3 段階
- メモは **任意**。やらないといけないものはタスクとして扱うべき。「できたらラッキー」の気持ち
- 読み返しに 2 つの壁: **タイミングの壁** と **解読の壁**
- 解読の壁を超える方法は唯一 **そもそもすぐ理解できるように書く**
- 行動指針は **Act first, Manage second** — できるなら行動、無理ならタスク管理に回す
- 処理は捨てる一択（例外あり）

## 章で導入される主な概念
- [メモ](memo.md) — オルタスク「メモ」の本格定義

## 引用しておきたい箇所
- 「**メモはスキル**」
- 「**Act first, Manage second**（可能なら行動してしまう、無理そうならタスク管理に回す）」
- 「**行動とは実質的にはタスク管理（ツール）に追加すること**」
- 「**自分が置かれた文脈の数だけ環境を用意しましょう**」
- 「行動し終えたメモはただのゴミ」
