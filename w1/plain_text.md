# プレーンテキスト (Plain Text)

装飾を持たない、素の文字情報。デジタル上で読み書きを扱う際の共通言語。

文字サイズ・色・リンクなどの装飾はツール固有のため他のツールに通用しないが、プレーンテキスト部分は共通言語のためどこでも通用する。

## 押さえるべき 2 点
- デジタル上で読み書きする際の素材は「プレーンテキスト」
- プレーンテキストは装飾なしの素の文字情報

## なぜ重要か

アナログの紙とペンは「自由自在」に書ける（紙面の好きな場所、打ち消し線、強調）。デジタルでも、プレーンテキストの仕様を知れば同じ自由自在を得られる。デジタルには操作の早さ、修正のしやすさ、データ同期のメリットがある（[アナログ/デジタル](analog_digital.md)）ため、プレーンテキストを習得することは「アナログの専売特許だった自由自在を、デジタルで実現する」近道となる。

## 仕様

### 行を並べる世界
プレーンテキストは行を並べることしかできない。アナログのような二次元的な書き込みはできない。

そのためタスク管理を行う際も「**1 行 1 タスク**」のようなルールで運用されることが多い。

### 使える文字は様々
ひらがな / カタカナ / 漢字 / 半角・全角・タブの空白 / 記号 / 絵文字（✅ 👍 🐰 など）。

タスク管理では単調な見栄えに **アクセントをつけてわかりやすくする** のが肝。たとえば完了表現は人それぞれ:

```
✅ごみ捨て
✅ ごみ捨て
[X] ごみ捨て
X ごみ捨て
xごみ捨て
```

正解はなく個人の好みになる。

### 保存する単位も様々
何らかの形で保存が要る。テキストエディタなら **ファイル**（`task.txt` など）。アプリなら **プロジェクト / ワークスペース / ページ / ファイル / ノート / ドキュメント** といった単位。

タスク管理を行う場合、この保存単位の設計（1 ファイルにまとめる / 月単位 / タスク単位）も意識する。

### カーソルを中心に操作する
プレーンテキストの操作はすべてカーソル位置を起点に行われる。文字の入力・消去・コピペ・表示範囲の追従もカーソルが基点。

> プレーンテキストの操作はカーソル操作をいかに上手くやれるかにかかっている

アナログでペンを所定位置に持っていくのに相当するのがカーソル操作。手慣れているとペンのように難なく扱えるが、原始的な矢印キー連打しかできないと消耗しやすく形骸化に繋がる。

### デジタル特有の操作
- **複製**（コピペ）が容易
- **検索** が一瞬（アナログは目視）
- **失敗してもすぐ消せる**

これらにより「一日数十のタスクをすべて管理する」レベルの運用も可能になる。

## 5 つの道具

プレーンテキストを扱うための道具は 5 種類。

| # | 道具 | 役割 |
| --- | --- | --- |
| 1 | **IME**（文字入力エンジン） | 文字入力・変換 |
| 2 | **エディットツール** | プレーンテキストの編集 |
| 3 | **ファイル形式と記法** | 何の流儀で書くか |
| 4 | **情報管理ツール** | 保存単位の管理 |
| 5 | **ヘルパー** | 入力支援 |

### 1: IME（文字入力エンジン）
Input Method Editor。バックグラウンドで動く文字入力プログラム。MS-IME（標準）/ Google 日本語入力 / ATOK が主要 3 つ。最近は性能差が小さいので **標準で十分**。

意識すべきは **辞書登録**。よく使う単語をカスタム登録すると便利。

著者の例:
- 「だん」で「✅」（Done から）
- 「ぼうめいたい」で「忘迷怠」（独自用語の登録）
- 「になって」で「担って」が出ないようにする（Google 日本語入力の抑制単語機能）

### 2: エディットツール

3 種類ある:

| 手段 | 動作の軽さ | オフライン | エディタ機能 | 習熟しやすさ | カスタマイズ性 |
| --- | --- | --- | --- | --- | --- |
| **テキストエディタ** | ⭕ | ⭕ | ⭕ | 🔺 | ⭕ |
| **IDE** | 🔺 | 🔺 | ⭕ | ❌ | ⭕ |
| **ウェブアプリ** | 🔺 | ❌ | ❌ | ⭕ | ❌ |

- **テキストエディタ** — プレーンテキストの編集に特化。動作が軽く、オフラインで動く。プレーンテキストでタスク管理する際にまず考えたい
- **IDE** — 統合開発環境。検索・ファイル管理など周辺機能が充実。デファクトは **Visual Studio Code (VSCode)**。元はプログラマー向けで習熟コストは高い
- **ウェブアプリ** — オンライン前提。チャット / ノート / Wiki / ブログ / SNS / BTS など。オンラインの遅延・接続切れが致命的なネック

> プレーンテキストでタスク管理する場合、ウェブアプリは **論外**。テキストエディタか IDE のいずれかを使う

ウェブアプリの問題は、軽微な遅延や接続不良が **アットランダムで発生** すること。プレーンテキストをガシガシ操作している最中の数秒の遅れでも、車で前方が急に減速するようなストレスになる。

家のたとえ: テキストエディタは **田舎の家**（DIY を頑張る、小回り）、IDE は **都会の家**（道具やサービスを駆使、習熟コストはかかる）。

### 3: ファイル形式と記法

**ファイル形式** = プレーンテキストをどんなファイルで扱うか:

- `task.txt` — テキスト形式（汎用、自然言語の比喩）
- `task.md` — Markdown 形式（マークアップ言語）
- `task.json` — JSON 形式（機械用、人間が読み書きするものではない）

**記法** = どんな書き方ルールに従うか。Markdown 記法 / JSON 記法など。記法をファイル拡張子で示したのがファイル形式。

**マークアップ言語** とは: 「こう書けば、こう表示する」と事前ルール化された記法。書くときはプレーンテキストで、読むときはプログラムが装飾をつけて見やすく表示する仕組み。

```
# 大見出し          ← Markdown
<h1>大見出し</h1>   ← HTML
```

Markdown は **軽量マークアップ言語** と呼ばれる。HTML より人間が書きやすい。

エディタは拡張子ごとに表示設定を変える。`.md` なら見出しを強調・目次を自動生成、など。独自記法 `.sta` を作ることもできる（自分で表示設定を作る必要があるので高度）。

### 4: 情報管理ツール

ファイルなど保存単位の管理。アプローチは 3 つ:

| アプローチ | 内容 |
| --- | --- |
| **使わない** | 保存単位の中で構造をつくる。例: `task.txt` 内に `■2024/07/30` のエリアをつくる。[Monolith](monolith.md) はこのアプローチ |
| **ファイラー** | 標準のエクスプローラー / Finder、VSCode のファイラー、Tablacus Explorer など |
| **ウェブアプリ** | クラウドに保存。ページ / ノート / ワークスペース等の単位 |

> 情報単位間（たとえばファイル間）を行き来するコストは高い

なるべく「使わない」アプローチが望ましい。

ファイラーを使う場合は **整理整頓の営みから逃れられない**。ただし、1 フォルダにフラット配置するなどシンプルな運用を確立できれば回避できる（年 365 個のファイルでも、整理しなければ整理は不要）。

### 5: ヘルパー
プレーンテキストを書く支援ツール:

- **Text Expansion** — `d[[` と入力すると今日の日付に展開、など。短縮語からスニペットを挿入
- **クリップボード履歴** — Win + V（Windows 10 標準）
- **アウトライン** — 見出しから目次を自動生成。VSCode + Markdown で標準利用できる
- **OCR** — 画像内文字をテキスト化（Gyazo Pro、PowerToys の Text Extractor など）

要するに **一から手打ち / 原始的なスクロールで探すといった手間を減らす** ための道具。

## おすすめの道具構成

| 道具 | おすすめ |
| --- | --- |
| IME | 何でも良い。**辞書登録** が重要 |
| エディタ | **テキストエディタか IDE**。ウェブアプリは外す |
| 形式・記法 | まずは **`.txt`** で自分の癖を反映、慣れてきたら Markdown |
| 情報管理 | まずは **単一ファイル**（[Monolith](monolith.md) 的）。増えてから考える |
| ヘルパー | 無くても OK。余力があれば一つずつ |

## アナログの仕様 vs プレーンテキストの仕様

| | アナログ | プレーンテキスト |
| --- | --- | --- |
| 単位 | 二次元的な紙面 | 行 |
| 操作起点 | ペン | カーソル |
| 残し方 | 自動的に紙に残る | 保存が要る |
| 複製 | 困難 | 容易（コピペ） |
| 検索 | 目視 | 一瞬 |
| 修正 | 消しゴム / 打ち消し | 即削除 |

両者は仕様が異なるだけで、慣れれば同様に「自由自在」を実現できる。
