# 「振り返り」

chap4 の 5 つ目。タスク管理の戦略を考えるための補助的活動として「振り返り」を扱う章。

## 要約

- **[振り返り](furikaeri.md)** = 過去の情報をインプットに考察を行い、アウトプット（タスク / モットー / コンテキスト）を出すこと
- 4 要素を **PROC** で整理: Purpose / Remembering / Opportunity / Continuation
- 定期的に行うのが鉄板。**DWMY**（Day/Week/Month/Year）と **DWMY ヒエラルキー**（一段下のアウトプットをインプットにする）
- 振り返りはタスクではなく **予定** として扱う。スタートラインに立てない人は外部の力（場所、もくもく会）を借りる
- 目的は 3 つに大別（**KAP**）: 知るため / 進むため / 守るため
- **4R マトリックス**: 客観/主観 × 自分/他 で 4 パターン（Reassessment / Review / Reflection / Recollection）
- アウトプットしたタスクが消化できないと **タスクデット** が積み上がる
