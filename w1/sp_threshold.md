# SP 臨界点（ソースとプランの臨界点）

[ソースアプローチ](task_source.md) から **プランアプローチ** に切り替えるベストタイミング。著者の造語。

## 2 つのアプローチ

| アプローチ | 性質 |
| --- | --- |
| **ソースアプローチ** | [タスクソース](task_source.md) から適宜タスク化。マイペースで疲弊小、本質や率直な議論が見えやすい。**探索的** |
| **プランアプローチ** | 計画やスケジュールを組んで動く。管理上都合がいいので現代仕事の主流。**計画的** |

両方使うのが現実的:

> 最初はソースアプローチを行ない、どこかでプランアプローチに切り替える

切り替えタイミングが SP 臨界点。

## 戦略の 4 パターン

| 戦略 | 説明 |
| --- | --- |
| **1: ソースのみ** | 締切なし、半永久的に続ける事柄。維持事項向き。テコ入れが必要なら部分的にプラン投入 |
| **2: プランのみ** | 現代ビジネスの自然な戦略だが VUCA な状況では悪手。建築・製造など単純な世界、規模の大きなプロジェクト・組織には有効 |
| **3: ソース → プラン** | **SP 臨界点を経由**。最も汎用的 |
| **4: プラン → ソース** | プロジェクト中止のときなど。残りをマイペースに |

## SP 臨界点の見極め

> 大体わかってきた、あとはいけそう

ソースアプローチであれこれこなしていくうちに情報や学びが増え、どこかで「見えてきた」と感じるポイント。ここでプランアプローチに切り替えれば、何もわからない状態から計画するよりはるかに現実的かつ無理のないものになる。

## SP 臨界点が最初から見えるケース

- **単純な世界**（作業に帰着できる仕事）
- **プロ** など著しく実力があって即見えてしまう

この場合は最初からプランアプローチで OK。ただしプランには余裕を持たせたい（交渉や日頃の[盤外戦](bangaisen.md)）。
