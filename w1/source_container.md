# 「コンテナ」

[オルタスク](altask.md) のひとつ「入れ物」を本格定義する章。タスク管理における **分類** を扱う。

## 要約

- [コンテナ](container.md) はタスクを分類したもの、または分類の各項目
- **リッチコンテナ** と **プレインコンテナ** で機能の豊富さで分かれる
- **箱モデル**（一つの箱に入れる）と **接続モデル**（目印をつける、複数 OK）の 2 つの世界観
- コンテナの扱い方は 4 段階（ツール通り → 独自運用 → ツール非依存 → 何も使わない）
- コンテナとタスクの違いは **粒度**。コンテナは粒度の粗いタスク
- 主な用語: フォルダ / カテゴリー / グループ / タグ / ラベル / プロジェクト / ワークスペース
- 応用: 大中小タスク、タスクと情報の仕分け、[コンテキストタグ](context_tag.md)、セクション、**コンテナのパピプペポ**

## 章で導入される主な概念
- [コンテナ](container.md) — オルタスク「入れ物」の本格定義
- [コンテキストタグ](context_tag.md) — 文脈（Context）とは別概念

## 引用しておきたい箇所
- 「**コンテナとは粒度の粗いタスク**」
- 「**タスクだとわかった時点ですぐにコンテナに入れる**」
- 「自分に合った分類を自分でつくるのが基本」
- 「分類が絶対的ではないこと、自分なりにやればいいこと、何ならやらなくてもいいこと」
