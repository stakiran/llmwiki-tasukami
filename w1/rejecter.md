# リジェクター (Rejecter)

拒否家。拒否したいタスクをあらかじめ決めておき、そのようなタスクが来たら問答無用で拒絶する [スタンス](stances.md)。省力化する系。

## Minimize Negatives を地で行く

「やらないこと（Not TODO）リスト」の発想。

- **Maximize Positives** — ポジティブなことを増やす（一般的）
- **Minimize Negatives** — ネガティブなことを減らす（リジェクター）

> 現代人は通常ネガティブを抱えすぎ、捨てるのが下手なので、Minimize Negatives はやりすぎるくらいがちょうど良かったりする

## 拒否ルール

「A が起きたら B をする」と具体的に条件と行動を定義したものを **拒否ルール** と呼ぶ。やらないことリストにテキトーに書くだけでは定着しないので、ちゃんと書く。

例:

- **見覚えのない人からのインターホンが来たら、居留守する** — 高確率でセールス・宗教勧誘・不審者
- **SNS で不快な物言いをする人がいたら、ミュートやブロックをする** — X のミュート、ブロック機能
- **17 時以降に通知が来ても、無視する** — 仕掛けが必要（毎日 16:50 / 7:30 に通知 ON/OFF 予定）

## リジェクターの 4T

|  | 反応的 | 予防的 |
| --- | --- | --- |
| スルーする | 1 反応的スルー | 2 予防的スルー |
| ブロックする | 3 反応的ブロック | 4 予防的ブロック |

セオリー: スルー → ダメならブロック → ダメなら離れる。**Through → Tackle → Transfer = 3T**。

厳密には 4T:

- **Through** — 様子見・我慢
- **Temporize** — スルー（仕組みで間接的に備える、張り紙やミュート）
- **Tackle** — ブロック（直接働きかける）
- **Transfer** — 自分から距離を置く・逃げる

リジェクターは **Temporize と Tackle を工夫する** スタンス。

## 反応的 vs 予防的

- 反応 — 対象が来る度に対応
- 予防 — 対象が来なくなるよう干渉、または来ても自動処理（[オートマトン](automaton.md) と地続き）

予防はコスト高だが、対象が何度も来るなら中長期的に楽。予防は **知識とスキル** に左右される。普段使っているツールの設定を見ておくとリジェクターのヒントが眠っている。

## 拒否リスト と 許可リスト

- **拒否リスト**（Deny List / Block List / Black List）— リジェクターの発想
- **許可リスト**（Allow List / Safe List / White List）— [フォーカシスト](focusist.md) の発想
