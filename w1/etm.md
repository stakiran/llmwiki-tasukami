# ETM（探索的タスク管理）

**Exploratory Task Management**。基準に頼らず好き勝手に動いて視界を広げていくタスク管理。

[文芸的タスク管理（LTM）](ltm.md) をベースにし、その上に **探索的なアプローチ** を重ねて再構築したもの。

## 概要

| | 内容 |
| --- | --- |
| **メリット** | 自由と楽しさ |
| **目的** | 視界を広げる |
| **適用シーン** | Unknown Unknown（何がわからないかもわからない場面） |
| **手段** | [LTM](ltm.md) + [4 つの管理](etm_management.md) + [ブロックライティング](block_writing.md) |
| **規模** | 個人または 2〜5 人程度の少人数 |
| **有限化** | [スクイーズアウト](etm_tips.md#スクイーズアウト)（現状の自分で 100% ひねり出す） |

### 探索的 = No ABCD
**No ABCD** = 以下の 4 つを行わない:

| 略 | 英語 | 意味 |
| --- | --- | --- |
| A | **Assign** | アサインしない（誰かに割り当てない） |
| B | **Ball** | ボールを持たない・渡さない |
| C | **Consensus** | 合意を取らない |
| D | **Deadline** | 締切をつくらない |

正解もなければ基準もなく、どう動くかは自分次第。

### メリット: 自由と楽しさ
従うべき基準が無く、各自が好きに探索するため自由。

> 楽しさは重要

私たちは怠け者なので、楽しさは貴重なモチベーションになる。子供が意味もなくうろつき、おもちゃをいじるような根源的な楽しさが探索にはある。

もう一つのパワーは **納得感**。何をどの順番でどれだけ深堀りするかをすべて自分で判断するため、結果が優れなくても「すべて自分がやってきた」という確かな事実が残る。

### 目的: 視界を広げる
正解も事例もない、何がわからないかわからない場面を自分なりに照らす。「自分なりに」なので正解とは限らないが、見えていれば行動できる。見えてからは指向的アプローチに切り替えるのが望ましい。

### ダンジョン探索のメタファー
広大で不確かな地形を自分なりに探索 + マッピングして広げていくイメージ。完全制覇は不可能なのでどこかで切り上げる。集めたアイテムが戦利品となり、次の（指向的アプローチが通用する）勝負で使う。

## 指向的アプローチとの対比

従来は **指向的（Directional）アプローチ**——先に基準をつくってからそれに従うあり方。当たり前すぎて名前がついていなかったため、ETM の対比でレトロニムとして命名された。

指向的アプローチの構造的欠陥:

| 欠陥 | 内容 |
| --- | --- |
| **あそびがない** | 基準にとらわれて融通が利かない。管理者がボトルネック。ニンジン（ムチ・アメ・コエ）が必要 |
| **試行錯誤のペースが遅い** | 基準作成側と実行側のコミュニケーションコスト。VUCA 時代に不適 |

ETM はこのうち **あそびがない** をカバーする。試行錯誤のペースは「初動を上げる」程度。

> 指向的アプローチを否定しているわけではない

ETM は別の道具。両者を **ハイブリッド** で使う（まず探索 → 大体見えたら指向に切り替え）のが定石。

## ETM のルーツ

著者提唱のため歴史はない。

参考: [LTM](ltm.md) のノートテイキング + 知的生産（特に [『知的生産の技術』](https://www.amazon.co.jp/dp/B000J85FH4) — 自分なりに新しい情報をつくり、他者にもわかるように整理すること）。

## 情報単位

ETM では 4 つの単位を扱う。

| 単位 | 内容 |
| --- | --- |
| **ゴール** | カードを集めて自ずと「見えてきた」もの |
| **カード** | 役に立つトピック。一つの知識・見解・意思決定・想像・仮説 |
| **トピック** | 一つの話題を表現したノート（[トピック指向](network_note.md#トピック指向)） |
| **ブロック** | トピック内の構成要素。特定の観点でまとめた塊 |

### ゴール

> カードを集めていくうちに自ずと見えてくる、あるいは執念でひねりだすもの

- 前もって見えているものではない
- 仮説として設定するものでもない
- 同じカードでも人や時期によって見えるゴールは違う
- 単一とは限らず、複数出ることもある

### カード = 役に立つトピック
トピックが 100 個あっても、カードとして使えるのは 20 個・30 個・5 個など。粒度は細かめ、枚数は数百以上になることも珍しくない。「パーツを揃える」「ボトムアップに積み上げる」感覚。

### トピックとブロックの関係

> 1-トピック N-ブロック

トピックは複数のブロックを含む。ただし:

- **論理的とは限らない** — 4 ブロックを使ってトピックを導いたとは限らない
- **ボトムアップ / トップダウン両方ありえる** — ブロックからトピックを導く / トピックからブロックを洗い出す

ETM は **部品化（パーツ）** を意識する。ゴール・カード・トピック・ブロック、すべて部品単位で扱う。

## トピックの育て方: 広げる・詰める・導く

トピックに対して行える 3 操作。

| 操作 | 英語 | 内容 |
| --- | --- | --- |
| **広げる** | Spread | 発散。思いつきや洗い出しを出す |
| **詰める** | Detailize | 拡充。調査・検討・声集めなど中身を埋める |
| **導く** | Summarize and Distill | 何かを導き出す（要約 or 蒸留） |

複数のトピックにまたがって行き来しながら育てる。自由奔放に見えるが本質はこの 3 つ。

### 詰めるは事実と主観を区別する
事実ばかりだと意思の無い学者、主観ばかりだと根拠のない評論家。バランスは個々で模索するが、最低でも区別はできるようにする。

### 要約と蒸留

| | スタンス |
| --- | --- |
| **要約 (Summarize)** | 部品から忠実に導く。改変せずに全部使う |
| **蒸留 (Distill)** | 部品をヒント（**ダシ**）にする。アレンジ・取捨選択・新しい思いつきも入れる |

蒸留 = **ダシにする**。元ネタは [『知的生産の技術』](https://www.amazon.co.jp/dp/B000J85FH4):
> いわば本をダシにして、自分のかってなかんがえを開発し、そだててゆくというやりかたである……このやりかたなら、読書はひとつの創造的行為となる

ETM では本だけでなくトピック・ブロック・他者の意見も全部ダシにできる。

### ブレイクスルー
蒸留による飛躍、特に肯定的な急進を **ブレイクスルー** と呼ぶ。

> ETM とはブレイクスルーを味わうための創造的な営み

要約で済むなら指向的アプローチで十分。それでは通用しない場面でこそ ETM を採用する。主観をフル活用してあらゆるものをダシにし本質をひねり出す。

## トピックを用いた知的生産

広げる・詰める・導くは別段珍しくない。

| 営み | 既存の手法 |
| --- | --- |
| 広げる | ブレインストーミング / フリーライティング / GTD の収集ステップ / 大喜利 |
| 詰める | 発散と収束 / Tak. のシェイク / 演繹と帰納 / ボトムアップとトップダウン |
| 導く | 意思決定 / プレゼンサマリー / ドメイン駆動開発の蒸留 |

3 つすべてをこなす手法として **KJ 法** がある。

ETM は知的生産の一手法にすぎない。LTM をベースにし、トピック・ブロックなどの概念を導入して再構成しただけ。

## ETM の管理

ETM におけるタスク管理は 5 つで構成される。実は **タスク管理以前の管理** がメイン。

| 管理 | 内容 |
| --- | --- |
| 1 ロード | 直近何を重視するか |
| 2 ターゲット | 直近どのトピックを重点的にいじるか |
| 3 リソース | 体力ややる気と上手く付き合う |
| 4 コンテキスト | あとで再開しやすいよう整理 |
| 5 タスク | 文芸的タスク管理に従う |

詳細は [ETM の 4 つの管理](etm_management.md) を参照。

## ワークフロー

3 ステップ。

### Step1: セットアップ
スコープと探索期間を決め、参加者と会場を準備する。

**スコープ** の条件:
- 絶対クリアしなければならないミッションがない
- 定量的な指標が含まれていない
- 達成できなくても存続が可能

**目指すゴール** はどうせ変わるので暫定で OK。「何か新規事業をつくる」くらいラフでよい。

**探索期間** = ETM をフルタイムで行う連続期間。「今日から二週間」など。
- 期間中は **指向的なアプローチを一切ストップ**
- 妥協案として午前 or 午後だけのハーフも可
- 長期休暇のようなもので、聖域

**探索時間** = 探索期間中の一日における探索時間。**最低 3 時間**。

> 探索期間という聖域は絶対に侵攻させてはなりません

管理者が「経過聞きたいから会議」も No ABCD 違反。会場に足を運んで情報収集すること。

#### メンバー
4 つの適性が必要:

- 自分ひとりでも読み書きしていける自律性
- LTM を行える程度の各種リテラシー
- 親しい相手でなくともコメントや意見を率直に書き込める何か
- 会話・会議に頼らずに過ごす耐性、または巻き込まず自分で解消する要領

#### ミュートデイ / ミュートウィーク
適性チェックの練習。

| 期間 | 単位 |
| --- | --- |
| **ミュートデイ** | 1 日間、誰とも喋らず通知を要する拘束も避ける |
| **ミュートウィーク** | 1 週間、同上 |

ステップアップ順:
1. ひとりでミュートデイ
2. ひとりでミュートウィーク
3. メンバー全員でミュートデイ
4. メンバー全員でミュートウィーク

#### 会場
[ネットワーク型ノートツール](network_note.md) 上のワークスペース / プロジェクト。Scrapbox の場合はプロジェクトを作成してメンバー招待。

### Step2: ETM
セットアップに従って ETM を行う。

**注意: ゴールはこのステップでつくる**。最後（Step3）ではなく道中で導く。ゴールも仮説検証の対象で何度も書き換える。

### Step3: ゴール判定
探索期間の終了時に振り返り。半日〜一日の会議。

判定内容:
- ETM を続けるかどうか
- 今後何をするか（方向性のみ、精度の高い計画は ETM 後に指向的アプローチで）

> ゴール判定で荒れたり長期化したらそもそも ETM が上手く行ってないか足りていない

短時間で済むはずだが、人は **節目を意識する生き物** なので時間を取る。残りは:

- **打ち上げ** — 数日確保しても OK。旅行など
- **自由時間** — 各自で休む・遊ぶ。仕事は入れない

両方ハイブリッドが無難（打ち上げは自由参加・途中入退場アリ）。

## 動的な ETM

3 ステップではなく、日々のワークスタイルに組み込む方法。

**探索モード** と **指向モード** を切り替える:

- 探索モード: ETM を行う。No ABCD
- 指向モード: 従来のコミュニケーションと管理

> 重要なのは両者を同時に行わないこと

切り替え戦略:

| 戦略 | 内容 |
| --- | --- |
| チーム単位で切り替え | 全員が探索 or 指向。混在を許さない |
| ひとりひとり必要に応じて切り替え | 5 人中 2 人が探索、3 人が指向もありえる |
| 人ごとに固定 | 常に探索の人と常に指向の人が分かれる |

切り替えは疲れるため、なるべく少ない方が良い。
