# 気付き: フレームワークは「事前計画ではなく事後の振り返り語彙」として使う

たすかみの厚いフレームワーク群（GSAAD / KAP / 4R / DWMY / PROC / 3T / 4T など）への、ゆるい知的生産的な使いこなし方。

## 主張

たすかみの略語フレームワークは、**事前に仕掛けようとすると重く、教える人が必要な手法に転落する**。

ゆるい知的生産の立場では、これらは:

- **最初から組み立てるための設計図** ではなく
- **試行錯誤した後で振り返るための語彙** として使う

「気付いたら GSAAD のループ通ってたな」「これ KAP の P だな」と **後付けで分解する道具** にとどめる。

## 出どころ

- [comment_habit_engineering.md](comment_habit_engineering.md):
  > GSAAD は事前計画ではなく事後の言語化に使う
- [comment_furikaeri.md](comment_furikaeri.md):
  > DWMY のリズム、KAP の目的分類、4R のマトリックス、PROC の検討要素、すべて 無くても回る。回らなくなったら導入する。

「フレームワークが厚すぎる」問題への対処として、**捨てる**（フレームワークを使わない）でも **被る**（フレームワーク通りに事前設計する）でもなく、**後ろにずらす** という第三の道。

## なぜこれが効くか

ストイック側は「先に枠組みを決め、それに従う」を推奨する。ゆるい知的生産の立場では、これは 2 つの問題を抱える:

1. **枠組み構築コスト** — 事前設計は重く、本来の作業時間を食う
2. **枠組み拘束コスト** — 一度決めた枠組みに引きずられ、当初想定外の道筋が見えなくなる

「事後の語彙」として使うと、両方を回避できる:

- 試行錯誤は **自由奔放** に行う（ブレストの本来モードと同じ）
- 結果が出てから「これは何の型だったか」を振り返って整理する
- 整理した結果は次の試行錯誤の **緩いガイド** にする（縛りではない）

これは [w1/ETM](../w1/etm.md) の「ゴールはあとから見えてくる」と同じ構造。**カードを集めるうちに自ずとゴールが見える** = 試行錯誤を続けるうちに自ずとフレームワークが当てはまる。

## 適用例

| フレームワーク | 事前計画モード（ストイック） | 事後語彙モード（ゆるい） |
| --- | --- | --- |
| **GSAAD** | 「これから散歩を習慣化するため Get-Set-Act-Adjust を組む」 | 散歩が続かなかった後で「Set が弱かったな、雨パラメーターを除外できてなかった」と振り返る |
| **KAP** | 「今期の振り返りは Advance を目的にする」と先に決める | 振り返り終わってから「これは Know だったな、Protect の要素もあったな」と分類する |
| **4R** | 「今日は Reflection をやる」と先に枠を取る | 書いたあとで「これ Recollection と Reflection が混ざってたな」と振り返る |
| **PROC** | 振り返り設計時に Purpose / Remembering / Opportunity / Continuation を全部詰める | 振り返り終わってから「Continuation を考えてなかったな」と気付く |
| **3T / 4T** | 「これは Tackle で対応する」と先に分類 | 対応後に「結局 Transfer が一番効いたな」と振り返る |

## 含意

- フレームワークは **棄てなくていい**。ゆるい知的生産はフレームワークと敵対しない
- ただし **使う場所を後ろにずらす** ことで、ストイック化を回避できる
- たすかみのような厚い本は、**読んで覚えておくだけで十分** で、覚えた語彙が振り返りのときに自然に立ち上がる
- これは [derived_self_awareness_signal.md](derived_self_awareness_signal.md) と整合する: 事前にフレームワークを語る = 自覚的、事後に語彙として使う = 半自覚

## 注意点

- 完全な事後運用が向かない場面もある:
  - **損失が大きい場面**（一度きりの大きな投資、健康に関わる行動、対人関係の決定的な選択）
  - これらは事前のフレームワーク化が要る
- ゆるい知的生産は **「重要度の低い試行錯誤」** に向いている。重要度が高いものは指向的アプローチ + 事前フレームワークが妥当
- たすかみの著者自身が言うように、ETM と指向的アプローチはハイブリッド

## 派生する問い

- 事前モードと事後モードの境界はどこか（重要度、可逆性、コストなど）
- 後付けで語彙を当てる訓練は、どこで身につくか（実践量？それともたすかみのような本の通読？）
- LLM が「あなたの今回の試行錯誤は GSAAD で言うとここに当たります」と事後に分類してくれるなら、人間は語彙を覚えなくてよくなるか
