# デリゲーター / スプリンター / スキッパーへのコメント

対象: [w1/デリゲーター](../w1/delegator.md), [w1/スプリンター](../w1/sprinter.md), [w1/スキッパー](../w1/skipper.md)

## 「抱える量を減らす」はゆるい知的生産のインフラ

3 スタンスはどれも **抱える量を減らす系**。これは [raw2/ゆるい知的生産.md](../raw2/ゆるい知的生産.md) で言う「ストイックではない」を、タスク量の側面から実装したものに当たる。

ゆるい知的生産は、頭の余白に「ぼんやり考える時間」と「井戸端的な雑談」が要る。抱える量が多いとその余白が消えるので、3 つのスタンスはすべて **ゆるい知的生産の物理的な前提条件** を整える役割を持つ。

## デリゲーター — 経済力ではなく立場で任せる

w1 から:

> 委譲できるかどうかは、委譲する・される立場（あるいは役割）があるかどうか で決まる
> — [w1/delegator.md](../w1/delegator.md)

> | 家事代行・執事など雇用 | お金の力で解決しているだけ。委譲は経済力に頼らず工夫や立場で誰かに任せる |
> — [w1/delegator.md](../w1/delegator.md) 「委譲のようで委譲ではないもの」

ここの定義はゆるい知的生産から見て少し厳しい。「経済力で解決＝委譲ではない」という線引きには疑問がある。独立した個人 SE にとって、家事代行や税理士や経理代行は **委譲そのもの** として運用するのが現実的。

ただし w1 の意図は理解できる。経済力で解決するだけだと、**「金が尽きたら委譲が消える」** 脆さがある。立場や役割に基づく委譲（共同経営、パートナー、信頼関係）は、金とは別軸でレジリエントなので、こちらが本物の委譲、という整理。

ゆるい知的生産の立場では、**両方使う** が正解。経済力での委譲（家事代行、税理士）と立場での委譲（パートナー、コミュニティ、OSS）を併用する。前者は単純な作業、後者は判断や文脈を要する作業。

## スプリンター — 認知資源の節約

w1 から:

> スプリンターの主目的は 認知資源をセーブすること
> — [w1/sprinter.md](../w1/sprinter.md)

ここは特に好き。「2 分ルール」「ボールはすぐ返せ」「フロントローディング」「毎日同じ服を着る」などの語彙はすべて、**判断という認知資源の浪費を避ける** 思想で繋がっている。

これは raw2 で言う「泥酔してても使えてほしい」と同じ場所を指している。判断を毎回しないで済む状態 = 認知資源を温存できている状態 = ゆるい知的生産に余白を回せる状態。

> 雑でもいいからとにかく進める
> — [w1/sprinter.md](../w1/sprinter.md) 「完璧じゃなくても前進する」

ここはブレストの「質より量」と完全に同型。即返信、即着手、即破棄、ぜんぶ「質より量」の運用形。ストイックに磨くより、雑に流して数を出す方がゆるい知的生産と相性がいい。

### LLM とスプリンター

LLM が出てきたことで、スプリンター戦略は **加速** した。

- メールの一次返信、Slack の一次返信、Issue の一次返信、ぜんぶ LLM が下書き
- コードレビュー、PR 説明文、リリースノート、ぜんぶ LLM が下書き
- 「2 分ルール」の閾値が下がる（実質は LLM に投げるだけで終わる）

スプリンターの「とにかく進める」は、LLM のおかげで「とにかく投げる」に降りてきた。ゆるい知的生産の立場としては、これは追い風。判断量が減るので余白が増える。

## スキッパー — デイリータスクを死守する裁量

w1 から:

> 先送りとは自分なりのペースを尊重するために（明日以降どこかでたぶんやらないといけないけど）今日はしない、と決めること
> — [w1/skipper.md](../w1/skipper.md) 「裁量の獲得が肝」

スキッパーは表面的には「楽そう」に見えるが、w1 自身が言う通り **野心的なスタンス**。デイリータスクを決めて、それ以外を撥ね除けるには、強い意志と裁量と説明責任の構えが要る。

ゆるい知的生産から見ると、スキッパーは **「今日のゆるさを死守する装置」** として機能する。今日の井戸端の時間、今日の散歩、今日のフリーライティング、これらを「外部からの依頼で侵食させない」ための仕組み。

> 今日はしない、と決められるだけの裁量が必要
> — [w1/skipper.md](../w1/skipper.md)

独立した個人 SE はここで詰むことが多い。顧客都合に合わせる癖が抜けないと、「今日はしない」が言えない。**裁量は獲得するもの** で、与えられるものではない。顧客選定、契約形態、料金体系、すべてが裁量の獲得装置。これは [スラキスト](comment_slackist.md) の「バッファを許す顧客／許さない顧客」と地続き。

## 3 スタンスのハイブリッドが現実

ゆるい知的生産を支持する個人 SE が、現実に運用するときは 3 つのハイブリッドになる:

- **デリゲーター** — 経理、税務、家事、雑務は外注。コミュニティに任せられるものは投げる
- **スプリンター** — 即返信できるものは即返信、特に LLM 経由で。判断を遅らせない
- **スキッパー** — それでも今日のデイリータスク（井戸端の時間、思考の時間、散歩、運動）は死守

3 つを同時に動かすと、抱える量が物理的に減って、ゆるい知的生産に必要な余白が確保できる。raw2 で言う「井戸端の方がアクティブ」状態を、自分の生活レベルで再現できる。

## アスリート系との温度感

w1 のスタンス分類で「抱える量を減らす系」は、[アスリート](../w1/athlete_applier.md) 寄りの気質と相性が良い。瞬発力で片付ける、即決即断、判断を増やさない。

ゆるい知的生産の立場としては、これは「**アスリート的な瞬発力で雑務を片付け、空いた時間でゆるく考える**」という二段構えに使える。一日のうちアスリートモードで雑務を 3 時間で片付け、残りはゆるく井戸端的に過ごす。混ぜないことで両立する。

ETM のコメントで書いた「動的 ETM の同時実行は無理」と同じ構造。**モードを分けるからゆるさが守れる**。
