# 気付き: 顧客 / パートナー選定軸としての「バッファ許容性 + トーカビリティ」

たすかみが「自分の素養」として書いているものを「相手を見る指標」に読み替えた整理。

## 主張

独立した個人 SE / 上級研究員にとって、**顧客・パートナー・共同経営者を評価するときの 2 軸** として、以下が使える:

| 軸 | 観察ポイント |
| --- | --- |
| **バッファ許容性** | 「一週間で終わる仕事に二週間ください」と言ったとき、理由を尋ねず受け入れるか |
| **トーカビリティ** | 定期的な対話の時間を取れるか、結論を急がず黙って考えられるか |

両方が高い相手とは長く付き合う。両方が低い相手とは、たとえ短期の利益があっても離れる。これがゆるい知的生産を **仕事の現場で守るための運用指標**。

たすかみは「自分が確保すべき素養」「自分が探すべきパートナー条件」として書いている。これを顧客評価マトリクスに転用したのがこの気付き。

## マトリクス

| | トーカビリティ高 | トーカビリティ低 |
| --- | --- | --- |
| **バッファ許容 高** | **理想**: 長期で付き合う。深い議論ができ、納期も柔軟 | バッファは取れるが議論が薄い。**作業特化の関係** |
| **バッファ許容 低** | 議論はできるが時間が無い。**短期の依頼を受ける程度に留める** | **避ける**: ストイックを強要してくる相手、長期では消耗 |

ゆるい知的生産を支持するなら、左上にだけ時間を配分する。右下から離れるための裁量を、契約・料金・キャラクター演出で確保する。

## 含意

### 観察方法
最初の数回の打ち合わせで判定可能:

- 「次回までに考えておきます」が機能するか、流されるか
- 沈黙を恐れず黙って考えるか、即座に何か喋ろうとするか
- 自分と違う意見が出たときに整理しようとするか、感情で反発するか
- 見積もりに対して「なんでそんなにかかるんですか」と即応するか、理由を聞くか

これらは **既に最初の打ち合わせで観察できる**。長く付き合ってから後悔するのではなく、入り口で判別する。

### 顧客選定は「営業のサボり」ではなく「ゆるい知的生産の自衛」
ストイック側の論調では「顧客は選ばず受けるのが営業の鉄則」とされやすい。だがゆるい知的生産の立場では、**顧客選定こそが盤外戦の主舞台**。

raw2 で suto3 が引いていた「自由奔放／質より量／結合改善／批判厳禁」は、ブレストを成立させる前提条件のリスト。これを仕事に翻訳すると、**前提条件を満たさない場（顧客）では、ゆるい知的生産は成立しない** ことになる。

### 私生活への拡張
仕事だけでなく、住む場所、付き合う人、所属するコミュニティにも同じマトリクスが効く。バッファが許される場所、対話が成立する場所、を選ぶ。

## 注意点

- この基準を厳密に運用すると、**選びすぎて取引相手がいなくなる** 危険がある
- 特に独立直後・キャリア初期は、左下や右上の相手から学ぶことも多い
- 「育てる」相手と「選ぶ」相手の使い分けが必要

つまりこの基準は **時期と立場による段階適用**。独立して数年経って実績ができてから、徐々に基準を厳しくしていくのが現実的。

## 派生する問い

- バッファ許容性とトーカビリティ以外に、ゆるい知的生産的な選定軸はあるか（候補: 即レス強制度、評価制度の柔軟さ、SaaS / 進捗管理ツールの押し付け度）
- LLM 時代に「トーカビリティ」は人間相手だけでなく LLM との対話にも拡張すべきか
- 自分がトーカビリティの高い相手として **選ばれる側** になるための条件は何か
