# 気付き: 振り返りで守るべき唯一のルールは「書き残すこと」

raw2 のブレスト本来ルール（提出されたアイデアを全部記録）を、振り返りに移植したら出てきた還元主義。

## 主張

たすかみの振り返りの章には、PROC / DWMY / DWMY ヒエラルキー / KAP / 4R / Enum and Assign / Diary Based DWMY / 5 分振り返りんぐ / トリガーリスト / MSCW など、多数のフレームワークがある。

しかしゆるい知的生産の立場では、これらは **すべて「あれば便利」** にすぎない。**唯一守るべきルール** は:

> **過去の自分が考えたことを、未来の自分が読み返せる場所に書き残す**

これだけ。これさえ満たせば振り返りはゆるく成立する。残りは全部オプション。

## 出どころ

- [raw2/ゆるい知的生産.md](../raw2/ゆるい知的生産.md):
  > ブレインストーミングの提唱者はこの 4 つのルールを「守るべきルールだ」とは言ってなくて、唯一守るべきルールは「提出されたアイデアをすべて記録すること」と言ってる
- [w1/ltm.md](../w1/ltm.md):
  > LTM ではノートを原本にする。頭の中はノートを読み込んで考えるための作業場にすぎない
- [comment_furikaeri.md](comment_furikaeri.md) — この気付きの初出

ブレストの「自由奔放／質より量／結合改善／批判厳禁」は守るべきルールではなく、唯一守るべきは「記録」だけ — という構造を、振り返りに当てはめた。

## なぜこれで足りるか

振り返りという営みの本質は **「過去 → 考察 → 次の行動」** だが、過去が消えていれば考察できない。**記録さえあれば、考察はあとからでも、何回でも、別の人格でもできる**。

逆に、PROC で目的を整理しても、KAP で目的分類しても、4R でマトリクス化しても、**書き残しが無ければ翌日には消える**。フレームワークは記録の代替にならない。

これは LTM の「ライティングスタックがすべての土台」と完全に同じ命題。ゆるい知的生産の振り返りは、要するに **LTM の運用そのもの** に還元できる。

## 具体的な「書き残し」の最低ライン

ゆるい知的生産的に運用するなら、書き残しはここまで軽くてよい:

- **その日に書いたノート / 触れたコード / 交わした会話** のうち、自分が引っかかったものを 1 行
- フォーマット不問。Slack の自分への DM、Scrapbox の今日の日付ページ、エディタの scratch ファイル、何でも
- 検索可能であること、後で読み返せること、これだけ満たす

これを満たせば、5 分振り返りんぐ も、Diary Based DWMY も、KAP も、必要なときに **後から立ち上げられる**。

## 含意

- たすかみの振り返り章は、**読んで覚えておけば十分** で、運用は記録だけで足りる（[derived_framework_post_hoc.md](derived_framework_post_hoc.md) と整合）
- 記録の形式が **泥酔してても書けるレベル** でないと続かない（raw2 の「泥酔してても使えてほしい」）
- LTM の本質は記録、振り返りの本質も記録、つまり **両者は同じ営み** とも言える
- LLM 時代には記録から自動で振り返りを生成できるが、**記録自体は依然として人間が書く必要がある**

## 注意点

- 「書き残しさえあればよい」は「**書き残し** が無ければ何もできない」と裏表
- 記録の習慣化は、記録ツール選定とハビットエンジニアリングが要る
- 記録が大量になると今度は読み返しコストが発生する → ここで LLM が効く

つまりこのルールは「フレームワークを捨てる」のではなく「**記録だけは絶対譲らない**」という最低防衛線の宣言。残りは状況に応じて足し引きする。

## 派生する問い

- 「書き残し」のフォーマットや場所は、人によってどこまで自由でよいか
- 書き残しを LLM に話すことで代替できるか（音声 → テキスト化）
- 「書きたくない日」をどう扱うか（=休む、=軽くする、=やめる）
