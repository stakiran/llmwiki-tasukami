# ゆるい知的生産から見た「たすかみ」

このページは **ゆるい知的生産を支持する個人のソフトウェアエンジニア** という立場から、たすかみ（[w1 Tasukami Wiki](../w1/index.md)）を読みながら気付きを書き溜めるための拠点である。

## 立場

- 独立した個人開発者、または企業内の IC / スタッフエンジニア / R&D の上級・主席研究者
- 自律性が高く、組織や社会との接点も自力で構築・メンテする
- 仕事もキャリアも文章書きも、いずれもソフトスキルが効く
- 知的生産は「ストイックに頑張るもの」ではなく「井戸端会議の延長で、楽にできてほしいもの」

## 元ネタ

[raw2/ゆるい知的生産.md](../raw2/ゆるい知的生産.md) — Scrapbox の議論ログ。要点を拾う:

- 知的生産に「ストイックな雰囲気」があるのは、半分は **手法を教えることを仕事にしている人** のせい、もう半分は **頑張ってる自分を自慢したい人** のせい
- 実際に知的生産ができている人（masui, shokai 等）は「頑張って階層整理できない」「泥酔してても使えてほしい」「どうすればもっと楽にできるのか」をずっと考えている
- ブレストの提唱者は「自由奔放／質より量／結合改善／批判厳禁」を **守るべきルールとは言っていない**。唯一守るべきは「提出されたアイデアをすべて記録すること」
- 「実際にやってる人がやってること」と「教える人が教えてること」が食い違う場合、信頼できるのは前者
- 普段使いの知的生産に小難しい理論はいらない（[/rashitamemo/普段使いの知的生産に小難しい理論はいらない](https://scrapbox.io/rashitamemo/普段使いの知的生産に小難しい理論はいらない)）

つまり **マナー講師がマナーを作る** 構造を疑え、というのが軸。

## たすかみとの相性

たすかみは厚い。3P / MCA / 3A / 3T / 4R / DWMY / KAP / PROC / GSAAD と略語が並ぶ。一見「ストイックの極み」に見える。

しかし読み込むと、著者自身が **盤外戦** や **アクティブレイジー** や **No ABCD** を提唱しており、「頑張らない／既存動線に乗る／自由に探索する」側にも厚く資源を割いている。むしろ「タスク管理が苦手な人や、ストイックになれない人」のための語彙が多い、と読める。

そのため、ゆるい知的生産を支持する立場としては **「たすかみは敵ではないが、選び取り方が要る」** が出発点になる。教科書ではなく [模索の道具箱](../w1/exploration.md) として読む。

## 視点

ゆるい知的生産の観点でたすかみを読むときの問いを 5 つ。

1. **泥酔しても使えるか** — その手法は、夜中に疲れて帰宅した自分でも使えるか
2. **教える人が必要な手法ではないか** — 解説書や講義なしでは扱えないなら、それは [マナー講師が作ったマナー](https://scrapbox.io/rashitamemo/普段使いの知的生産に小難しい理論はいらない) かもしれない
3. **既存動線に乗っているか** — 新しい仕組みを足していないか。Slack / メール / カレンダー / GitHub のように、既に使っているものの中で完結できないか
4. **「頑張ってる自分を自慢する」装置になっていないか** — その手法を見せびらかすことが目的化していないか
5. **記録だけは確保できているか** — ブレストの本来の唯一ルール「提出されたアイデアをすべて記録すること」が満たされているか
